## 概要

### TheOption Trend Analyzerとは？

**TheOption Trend Analyzer**は、バイナリーオプション取引プラットフォーム「TheOption」専用のAI支援型トレンド予測Chrome拡張機能です。

### 核心的な価値

従来のテクニカル分析ツールとは異なり、本システムは**「過去の類似パターンから未来を予測する」**という機械学習アプローチを採用しています。

**従来のツール**:
- RSI、MACD、移動平均線などの指標を表示
- トレーダー自身が指標を解釈して判断

**TheOption Trend Analyzer**:
- 現在の市場状況を多次元的に分析
- 過去の類似した状況を自動検索
- その時の結果から未来を予測
- 信頼度付きで予測を提示

## このシステムでできること

### 1. リアルタイム価格予測

**5つの時間枠で同時予測**
- 15秒後
- 30秒後
- 60秒後（1分後）
- 180秒後（3分後）
- 300秒後（5分後）

**予測内容**
- 方向性: HIGH（上昇）/ LOW（下降）/ NEUTRAL（中立）
- 信頼度: 50%〜100%（類似パターンの一致度）
- 類似データ数: 予測の根拠となったデータ件数

### 2. 多次元市場分析

**10種類のテクニカル指標を統合分析**
- RSI（相対力指数）
- 移動平均線（MA5, MA20）
- MACD（移動平均収束拡散法）
- ADX（平均方向性指数）
- ストキャスティクス
- ATR（平均真の範囲）
- ROC（変化率）
- ボリンジャーバンド
- 市場センチメント
- 時間帯バイアス

### 3. セグメントベースパターン認識

**革新的な「セグメント分析」**

価格の動きを細かく分割（セグメント化）して、形状パターンを認識します。

例: 60秒の価格履歴を10秒×6セグメントに分割
```
セグメント1: 上昇 (UP)
セグメント2: 下降 (DOWN)
セグメント3: 平坦 (FLAT)
セグメント4: 上昇 (UP)
セグメント5: 下降 (DOWN)
セグメント6: 上昇 (UP)

パターン: "U-D-F-U-D-U"
```

この形状パターンが過去データと一致するかを高速検索し、類似度を計算します。

### 4. 通貨ペア別の動的閾値調整

**各通貨ペアの特性に自動適応**

- **仮想通貨（BTC/USD）**: ボラティリティが高い → 閾値を自動的に高く設定
- **マイナー通貨（AUD/NZD）**: ボラティリティが低い → 閾値を自動的に低く設定
- **メジャー通貨（EUR/USD）**: 中間的なボラティリティ → バランスの取れた閾値

これにより、**全ての通貨ペアで最適な精度**を実現します。

### 5. 学習データの継続的な蓄積

**使えば使うほど賢くなる**

取引を続けるごとに：
- 市場状況データが自動保存
- 結果データ（15秒後、30秒後...の実際の価格）が記録
- 次回以降の予測精度が向上

**ストレージ容量**: 無制限（Chrome storage unlimited）

### 6. 視覚的なトレンド表示

**直感的なUI**
- 📈 **上昇トレンド**: 緑色の上矢印
- 📉 **下降トレンド**: 赤色の下矢印
- ⚖️ **中立**: グレーの横線
- **信頼度バー**: 視覚的な確信度表示

### 7. 類似度閾値の柔軟な調整

**4段階の閾値設定**
- 50%: より多くのパターンを許容（積極的な予測）
- 70%: バランス型（推奨）
- 80%: 高精度重視
- 90%: 超慎重型（確実な場合のみ予測）

### 8. 学習データのエクスポート

**4種類のCSVダウンロード**
1. **AI学習データ**: 全テクニカル指標 + セグメント分析 + 結果データ
2. **価格履歴データ**: タイムスタンプと価格の生データ
3. **予測履歴データ**: 過去の予測とその精度
4. **トレンド分析データ**: 市場トレンドの統計情報

## 内部システムの詳細

### 1. 多次元分析エンジン（Multi-Indicator System）

**ファイル**: `multi-indicator-system.js`

#### 役割
10種類のテクニカル指標を統合的に計算し、市場の多面的な状況を把握します。

#### 計算される指標

**1. RSI（Relative Strength Index）**
```javascript
期間: 14
範囲: 0〜100
解釈:
  - 70以上: 買われすぎ（下降の可能性）
  - 30以下: 売られすぎ（上昇の可能性）
```

**2. 移動平均線（MA5, MA20）**
```javascript
MA5: 直近5期間の平均
MA20: 直近20期間の平均
クロス:
  - ゴールデンクロス: MA5 > MA20（上昇シグナル）
  - デッドクロス: MA5 < MA20（下降シグナル）
```

**3. MACD（Moving Average Convergence Divergence）**
```javascript
計算:
  MACD線 = EMA12 - EMA26
  シグナル線 = MACD線のEMA9
強度 = |MACD線 - シグナル線|
```

**4. ADX（Average Directional Index）**
```javascript
範囲: 0〜100
解釈:
  - 25以上: 強いトレンド
  - 25未満: レンジ相場
```

**5. ストキャスティクス**
```javascript
%K = (現在価格 - 最安値) / (最高値 - 最安値) × 100
範囲: 0〜100
解釈:
  - 80以上: 買われすぎ
  - 20以下: 売られすぎ
```

**6. ATR（Average True Range）**
```javascript
ボラティリティ指標
ATR% = (ATR / 現在価格) × 100
用途: 動的閾値の計算に使用
```

**7. ROC（Rate of Change）**
```javascript
変化率 = (現在価格 - N期間前の価格) / N期間前の価格
短期的なモメンタムを測定
```

**8. ボリンジャーバンド**
```javascript
中心線 = MA20
上限 = MA20 + (標準偏差 × 2)
下限 = MA20 - (標準偏差 × 2)
位置 = (現在価格 - 下限) / (上限 - 下限)
```

**9. センチメントスコア**
```javascript
直近の上昇・下降の比率
上昇比率 = 上昇回数 / 総回数
範囲: 0〜1
```

**10. 時間帯バイアス**
```javascript
時間帯ごとの上昇・下降傾向を統計的に分析
朝: 上昇傾向
夜: 下降傾向
など
```

### 2. セグメント分析エンジン（Detailed Segment Analyzer）

**ファイル**: `detailed-segment-analyzer.js`

#### 革新的な「セグメント分析」とは？

従来の価格分析は「点」で見ていました：
```
現在価格: 1.08532
1分前: 1.08520
→ 上昇している
```

セグメント分析は「形状」で見ます：
```
60秒の価格履歴を6つのセグメントに分割

セグメント1 (0-10秒):  1.08520 → 1.08525 (上昇)
セグメント2 (10-20秒): 1.08525 → 1.08518 (下降)
セグメント3 (20-30秒): 1.08518 → 1.08519 (平坦)
セグメント4 (30-40秒): 1.08519 → 1.08528 (上昇)
セグメント5 (40-50秒): 1.08528 → 1.08520 (下降)
セグメント6 (50-60秒): 1.08520 → 1.08532 (上昇)

形状パターン: "U-D-F-U-D-U"
```

この「形状の指紋」を使って過去データを検索します。

#### セグメントの詳細分析

**各セグメントで計算される特徴量**:

```javascript
{
  // フェーズ1: 基本統計
  startPrice: 1.08520,
  endPrice: 1.08525,
  change: 0.00005,
  changePercent: 0.0046,
  avgPrice: 1.08522,

  // フェーズ2: トレンド特徴
  slope: 0.0041,              // 線形回帰の傾き
  direction: "UP",            // 方向性
  magnitude: 0.0046,          // 変化の強度
  volatility: 0.00002,        // ボラティリティ
  range: 0.00008,             // 価格幅
  rangePercent: 0.0074,
  upRatio: 0.7,               // 上昇の割合

  // フェーズ3: 形状特徴
  maxPrice: 1.08526,
  minPrice: 1.08518,
  peakPosition: 0.8,          // ピークの位置
  troughPosition: 0.2,        // ボトムの位置
  patternType: "UPTREND"      // パターン分類
}
```

#### 動的閾値システム

**問題**: 固定閾値では全通貨ペアに対応できない

**解決策**: ATRと統計を組み合わせた動的閾値

```javascript
// ATRベース閾値
atrBasedThreshold = ATR% × 0.1

// 統計ベース閾値
過去100件の変化率 → 標準偏差を計算
statsBasedThreshold = 標準偏差 × 0.25

// 加重平均
finalThreshold = atrBasedThreshold × 0.6 + statsBasedThreshold × 0.4

// 範囲制限
finalThreshold = clamp(0.001%, 0.1%)
```

**効果**:
- BTC/USD: 閾値 0.017%（高ボラティリティ対応）
- EUR/USD: 閾値 0.008%（中程度）
- AUD/NZD: 閾値 0.003%（低ボラティリティ対応）

### 3. 機械学習システム（Machine Learning System）

**ファイル**: `machine-learning-system.js`

#### データ収集システム

**役割**: 市場状況と結果を記録

```javascript
// 10秒ごとに実行
collectData() {
  現在の状況を記録 {
    timestamp: 現在時刻,
    price: 現在価格,
    indicators: 全テクニカル指標,
    segments: 全時間枠のセグメント分析
  }

  // 15秒後、30秒後...の価格を非同期で記録
  setTimeout(() => {
    15秒後の価格を記録
  }, 15000)

  setTimeout(() => {
    30秒後の価格を記録
  }, 30000)

  // ... 60秒、180秒、300秒後も同様
}
```

**保存タイミング**: 10件蓄積ごと

#### パターンマッチングシステム

**役割**: 類似パターンを高速検索

**ステップ1: セグメント類似度計算（40点満点）**

```javascript
// 現在のパターン
current = "U-D-F-U-D-U"

// 過去のパターン
historical = "U-D-U-D-U-D"

// 各セグメントの一致度
segment1: U ⇄ U → PERFECT (1.0)
segment2: D ⇄ D → PERFECT (1.0)
segment3: F ⇄ U → NONE (0.05)
segment4: U ⇄ D → NONE (0.05)
segment5: D ⇄ U → NONE (0.05)
segment6: U ⇄ D → NONE (0.05)

// 重み付けスコア計算
weights = [1, 2, 4, 8, 16, 32]  // 新しいセグメントほど重要
total = (1×1 + 2×1 + 4×0.05 + 8×0.05 + 16×0.05 + 32×0.05) / 63
score = 12.8 / 40点
```

**ステップ2: テクニカル指標類似度（30点満点）**

```javascript
// RSI類似度
rsiDiff = |current.rsi - historical.rsi|
rsiScore = max(0, 1 - rsiDiff/50) × 5点

// MACD類似度
macdScore = ... × 5点

// ... 他の指標も同様

total = 各指標スコアの合計 / 30点
```

**ステップ3: 時間的類似度（30点満点）**

```javascript
// 時間帯の近さ
hourDiff = |current.hour - historical.hour|
hourScore = max(0, 1 - hourDiff/12) × 10点

// 曜日の一致
dayScore = (current.day === historical.day) ? 10点 : 0点

// 分の近さ
minuteScore = ... × 10点

total = 各スコアの合計 / 30点
```

**ステップ4: 総合類似度**

```javascript
totalSimilarity = (
  segmentScore × 40 +
  indicatorScore × 30 +
  timeScore × 30
) / 100

// パーセンテージ表示
similarity = totalSimilarity × 100  // 例: 68.5%
```

**ステップ5: フィルタリング**

```javascript
if (similarity >= currentSimilarityThreshold) {
  このデータを予測に使用
} else {
  除外
}
```

#### 予測生成システム

**役割**: 類似データから統計的予測を生成

```javascript
// 類似データを集計
similarData = [20件の類似データ]

// 15秒後の結果を集計
results15s = similarData.map(d => d.result15s.direction)
// ["HIGH", "HIGH", "LOW", "HIGH", "HIGH", ...]

upCount = results15s.filter(r => r === "HIGH").length  // 15件
downCount = results15s.filter(r => r === "LOW").length  // 3件
flatCount = results15s.filter(r => r === "FLAT").length  // 2件

// 信頼度計算
upRate = upCount / 20 = 75%
downRate = downCount / 20 = 15%

// 予測
if (upRate >= 70%) {
  prediction = "HIGH"
  confidence = 75%
} else if (downRate >= 70%) {
  prediction = "LOW"
  confidence = downRate
} else {
  prediction = "NEUTRAL"
  confidence = null
}
```

**信頼度閾値**:
```javascript
CONFIDENCE_THRESHOLD = 70%

if (confidence >= 70%) {
  予測を表示
} else {
  "データ不足" と表示
}
```

### 4. セグメント類似度計算システム

**ファイル**: `segment-similarity-calculator.js`

#### EnhancedSegmentScoring（強化版セグメントスコアリング）

**革新的な点**: 新しいセグメントほど重視する

```javascript
weights = [1, 2, 4, 8, 16, 32]
// セグメント1: 重み 1
// セグメント2: 重み 2
// セグメント3: 重み 4
// セグメント4: 重み 8
// セグメント5: 重み 16
// セグメント6: 重み 32（最新・最重要）
```

**理由**: 最新の価格動向が未来に最も影響を与えるため

#### マッチレベルの定義

```javascript
PERFECT: 1.0    // 完全一致（UP⇄UP, DOWN⇄DOWN）
HIGH:    0.7    // 同じ方向性
MEDIUM:  0.4    // やや似ている
NONE:    0.05   // 不一致（UP⇄DOWN）
```

#### 低ボラティリティフィルター

**問題**: 全セグメントがFLATだと50-50の予測になる

**解決策**: アクティブセグメント数をカウント

```javascript
// アクティブセグメント判定
isActive(segment) {
  if (segment.direction === "UP" || segment.direction === "DOWN") {
    return true
  }
  if (segment.magnitude >= 0.05%) {  // FLATでも変化量が大きければアクティブ
    return true
  }
  return false
}

// パターン全体の活動度チェック
activeCount = segments.filter(s => isActive(s)).length

if (activeCount < 3) {  // 6セグメント中3つ未満
  // 低ボラティリティ → 類似度0を返す
  return { similarity: 0, lowVolatility: true }
}
```

**効果**:
- 全FLATパターンは除外される
- 意味のあるパターンのみが予測に使われる

## FAQ

### Q1: 本当に勝率は上がりますか？

**A**: 統計的には優位性があります。

**理由**:
- 市場にはパターンが存在する
- 類似した状況では類似した結果になりやすい
- 多次元的な分析で精度が向上

**ただし**:
- 100%の勝率は不可能
- 市場環境によって精度は変動
- データ量が多いほど精度向上

**期待できる勝率**:
```
データ100件:   55-58%
データ1,000件: 60-63%
データ10,000件: 65-68%
```

### Q2: どのくらいデータを集めれば使えますか？

**A**: 最低100件、推奨は1,000件以上

**データ収集時間の目安**:
```
1時間:   約360件（1通貨ペア）
1日:     約1,500件
1週間:   約10,000件
```

**推奨**:
1. 最初の1日: データ収集のみ
2. 2日目以降: 予測を参考に取引開始
3. 1週間後: 本格的に活用

### Q3: 全ての通貨ペアで使えますか？

**A**: はい、動的閾値により全通貨ペアに対応しています。

**対応通貨ペア**:
- メジャー通貨: EUR/USD, GBP/USD, USD/JPY
- マイナー通貨: AUD/NZD, EUR/GBP
- 仮想通貨: BTC/USD, ETH/USD

**注意点**:
- 通貨ペアごとに別々のデータが必要
- 流動性の低い通貨ペアは精度が落ちる可能性

### Q4: 類似度閾値はどれを選べば良いですか？

**A**: 状況に応じて使い分けてください。

**推奨設定**:
```
データ100件未満:  50%（積極的）
データ100-1,000件: 70%（バランス・推奨）
データ1,000件以上: 80%（慎重）
重要な局面:       90%（超慎重）
```

**トレードオフ**:
- 低い閾値: 予測数↑ 精度↓
- 高い閾値: 予測数↓ 精度↑

### Q5: データが増えすぎたらどうなりますか？

**A**: Chrome Storageは無制限です。

**ストレージ管理**:
- データは圧縮して保存
- 古いデータも保持（長期トレンドに有効）
- 10,000件で約5-10MBの使用量

**メンテナンス**:
- 基本的に不要
- 気になる場合は月1回のCSVエクスポート推奨

### Q6: 他のテクニカル分析ツールと併用できますか？

**A**: はい、併用を推奨します。

**推奨の組み合わせ**:
```
このシステム: パターンベースの予測
   +
TradingView: チャート分析、サポート/レジスタンス
   +
経済指標カレンダー: ファンダメンタルズ分析
```

**相乗効果**:
- このシステムの予測 + チャートの確認
- 重要指標発表前は取引を控える
- 複数の分析が一致した場合、信頼性↑

### Q7: スマートフォンでも使えますか？

**A**: 現時点ではPC版Chromeのみ対応です。

**理由**:
- Chrome拡張機能の制限
- モバイル版Chromeは拡張機能非対応

**代替案**:
- iPadなどのタブレットでPC版サイトを表示
- リモートデスクトップでPCに接続

### Q8: データは安全ですか？

**A**: 全てローカルストレージに保存されます。

**セキュリティ**:
- データは端末内のみ（外部送信なし）
- Chrome Storage Local使用
- 暗号化された保存

**プライバシー**:
- 個人情報は収集しない
- 取引履歴は記録しない
- 価格データと分析結果のみ

### Q9: 予測が外れたらどうなりますか？

**A**: 外れたデータも学習に使われます。

**学習サイクル**:
```
予測: HIGH (信頼度75%)
結果: 下降
   ↓
このパターンは「下降になる場合もある」と学習
   ↓
次回同じパターンで信頼度↓
   ↓
予測精度が向上
```

**重要**: 外れることも含めて学習システム

### Q10: システムを信じて全資金を投入しても大丈夫ですか？

**A**: 絶対にやめてください。

**リスク管理の鉄則**:
```
1取引あたり: 資金の2-5%以下
1日あたり: 資金の10%以下の損失で停止
連続3敗: その日の取引を停止
```

**このシステムの位置づけ**:
- あくまで「支援ツール」
- 最終判断は人間が行う
- 100%の勝率は存在しない

**心構え**:
- システムを盲信しない
- 他の分析も併用する
- 感情的な取引をしない
- 損失許容範囲を守る
