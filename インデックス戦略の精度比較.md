# インデックス戦略の精度比較

## 重要な前提

**両方とも最終的には全7指標で詳細計算する**

```javascript
// どちらの戦略でも最後は同じ処理
絞り込んだ候補（40-100件）
  ↓
全7指標で詳細な類似度計算
  ↓
類似度でソート → 上位20件で予測
```

→ **理論的には同じ精度** ✅

---

## しかし、実際には違いが出る理由

### 理由1: 絞り込みで「良いデータ」が除外される可能性

#### 例：本当に似ているデータが除外されるケース

**データX（過去のデータ）**:
```javascript
{
  segment: "UP-UP-UP-DOWN-DOWN-DOWN",  // 現在と完全一致
  rsi: 68,      // "買われすぎ" ← 現在と一致
  macd: 0.012,  // "横ばい" ← 現在（0.025="上昇"）と不一致！
  roc: 0.4,
  stochastic: 75,
  adx: 38,
  // 詳細計算すると類似度90点（非常に似ている）
}
```

**現在の相場**:
```javascript
{
  segment: "UP-UP-UP-DOWN-DOWN-DOWN",
  rsi: 75,      // "買われすぎ"
  macd: 0.025,  // "上昇"
  roc: 0.5,
  stochastic: 78,
  adx: 40
}
```

#### 戦略2の挙動

```javascript
// 主要3指標で絞り込み
segment = "UP-UP-UP-DOWN-DOWN-DOWN" ✓
rsi = "買われすぎ" ✓
macd = "上昇" ✗  ← データXは"横ばい"

→ データXは候補から除外される ❌
→ 詳細計算される機会がない
→ 本当は類似度90点だったのに！
```

#### 戦略3の挙動

```javascript
// 複合指標で絞り込み
segment = "UP-UP-UP-DOWN-DOWN-DOWN" ✓

momentum = classify(rsi=68, stochastic=75)
         = "買われすぎ" ✓

trend = classify(macd=0.012, roc=0.4, maCross=...)
      = "上昇寄り" ✓  ← 複合的に判定するので許容範囲

strength = classify(adx=38)
         = "強い" ✓

→ データXは候補に残る ✅
→ 詳細計算で類似度90点と判明
→ 予測に使用される
```

**結果**: 戦略3の方が良いデータを逃さない可能性がある

```javascript
  大半が除外
```


## なぜ戦略3の方が精度が高いか

### 理由1: 柔軟な判定

```javascript
// 戦略2（厳格）
RSI = 72
→ "買われすぎ"（70-100）
→ RSI=69のデータは除外される ❌

// 戦略3（柔軟）
momentum = classify(RSI=72, Stochastic=75)
→ "買われすぎ"
→ RSI=69でもStochastic=78なら「買われすぎ」に分類 ✅
```

### 理由2: 相互補完

```javascript
// 戦略2
MACD = 0.018 → "横ばい"
ROC = 0.5 → "上昇"
→ 2つの独立した条件

// 戦略3
trend = classify(MACD=0.018, ROC=0.5, MACross=...)
→ MACDは微妙だがROCとMACrossが上昇
→ 総合的に"上昇"と判定 ✅
```

### 理由3: 境界値問題の回避

```javascript
// 戦略2の問題
RSI = 69.9 → "中立"
RSI = 70.1 → "買われすぎ"
→ わずか0.2の違いで別カテゴリ ❌

// 戦略3
momentum = classify(RSI=69.9, Stochastic=80)
→ RSIは境界だがStochasticで補完
→ "買われすぎ"に分類される ✅
```
