# ストレージ容量確認方法

## 修正内容

### 1. unlimitedStorage パーミッション追加
`manifest.json` に `unlimitedStorage` パーミッションを追加しました。
これにより、Chrome の storage.local の10MB制限が解除されます。

### 2. データ制限を元に戻す
- メモリ内データ制限: 60,000件（50,000件にサンプリング）
- ストレージ保存制限: 50,000件

### 3. データサイズ表示機能追加
保存時に自動的にデータサイズをコンソールログに表示します。

---

## ストレージ容量の確認方法

### 方法1: コンソールログで自動確認（推奨）

拡張機能がデータを保存するたびに、以下のようなログが表示されます：

```
[ML] 💾 USD/JPY: 6870件のデータを保存しました（元: 6870件）
[ML] 💾 データサイズ: 1234.56 KB (1.21 MB)
```

**確認ポイント:**
- `データサイズ` の値が表示されます
- 複数通貨ペアを使用している場合、通貨ペアごとに表示されます

---

### 方法2: DevTools で Chrome ストレージを直接確認

1. **Chrome拡張機能の管理画面を開く**
   - `chrome://extensions/` にアクセス
   - 「TheOption Trend Analyzer」を探す

2. **DevTools を開く**
   - 拡張機能の「詳細」をクリック
   - 「バックグラウンドページ」または「Service Worker」のリンクをクリック
   - DevTools が開きます

3. **Application タブを開く**
   - DevTools の「Application」タブをクリック
   - 左側のメニューから「Storage」→「Local Storage」を選択

4. **容量を確認**
   - `theoption_ml_*` のキーを確認
   - 各キーのサイズが表示されます

---

### 方法3: コンソールでスクリプト実行

TheOption のページのコンソールで以下のスクリプトを実行：

```javascript
// 全ストレージキーを取得して容量を表示
chrome.storage.local.get(null, (items) => {
  let totalSize = 0;
  let totalSizeMB = 0;

  console.log('========== ストレージ容量詳細 ==========');

  for (const [key, value] of Object.entries(items)) {
    if (key.startsWith('theoption_ml_')) {
      const str = JSON.stringify(value);
      const sizeKB = (str.length / 1024).toFixed(2);
      const sizeMB = (str.length / 1024 / 1024).toFixed(2);
      const count = Array.isArray(value) ? value.length : 'N/A';

      console.log(`${key}:`);
      console.log(`  - データ件数: ${count}件`);
      console.log(`  - サイズ: ${sizeKB} KB (${sizeMB} MB)`);

      totalSize += str.length;
    }
  }

  totalSizeMB = (totalSize / 1024 / 1024).toFixed(2);
  console.log('========================================');
  console.log(`合計サイズ: ${totalSizeMB} MB`);
  console.log(`unlimitedStorage: 有効 (制限なし)`);
});
```

**出力例:**
```
========== ストレージ容量詳細 ==========
theoption_ml_USD_JPY:
  - データ件数: 6870件
  - サイズ: 1234.56 KB (1.21 MB)
theoption_ml_EUR_USD:
  - データ件数: 4500件
  - サイズ: 800.12 KB (0.78 MB)
========================================
合計サイズ: 1.99 MB
unlimitedStorage: 有効 (制限なし)
```

## 容量が問題ないかの判断基準

### ✅ 正常な状態
- `unlimitedStorage` パーミッションが有効
- コンソールに `QuotaExceededError` が表示されない
- データサイズログが正常に表示される
- `[ML] 💾` ログに「Chrome storage エラー」が表示されない

### ⚠️ 注意が必要な状態
- 1通貨ペアあたり 5MB 以上のデータ
- 複数通貨ペアで合計 50MB 以上
- メモリ使用量が高い場合（ブラウザが重い）

### ❌ 問題がある状態
- `QuotaExceededError` が表示される
- 「Chrome storage エラー」ログが表示される
- データが保存されない

---

## トラブルシューティング

### 問題: QuotaExceededError が発生する

**原因:** `unlimitedStorage` パーミッションが適用されていない

**解決方法:**
1. Chrome拡張機能を**完全に削除**
2. 再度インストール（または「拡張機能を読み込む」）
3. 拡張機能の詳細で「権限」を確認し、「unlimitedStorage」が表示されているか確認

**注意:** 単に「再読み込み」ではパーミッションが更新されない場合があります。

---

### 問題: データサイズが大きすぎる

**対策1: データ件数を減らす**

`machine-learning-system.js` の以下の行を編集：
```javascript
// 60000件 → 30000件に変更
if (this.trainingData.length > 30000) {
  this.trainingData = this.intelligentSampling(this.trainingData, 25000);
}

// 50000件 → 25000件に変更
const dataToSave = this.intelligentSampling(this.trainingData, 25000);
```

**対策2: 古いデータを削除**

コンソールで実行：
```javascript
chrome.storage.local.remove(['theoption_ml_通貨ペア名'], () => {
  console.log('データを削除しました');
});
```

## 定期的な監視

以下の頻度で容量をチェックすることを推奨：
- **初期段階（最初の1週間）**: 毎日
- **安定稼働後**: 週1回
- **複数通貨ペアを追加した時**: 即座に確認

---

## 参考情報

- **Chrome Storage API 制限（unlimitedStorage なし）**: 約10MB
- **unlimitedStorage パーミッション**: 実質無制限（PCのディスク容量次第）
- **推奨データサイズ**: 1通貨ペアあたり 5MB 以下（パフォーマンスのため）
- **最大データ件数**: 50,000件（パフォーマンスと精度のバランス）

---

## まとめ

1. **自動ログで確認**（最も簡単）
   - `[ML] 💾 データサイズ:` のログを見る

2. **手動スクリプトで詳細確認**（定期チェック用）
   - 上記のスクリプトをコンソールで実行

3. **問題が発生したら**
   - 拡張機能を再インストール
   - データ件数を調整

unlimitedStorage パーミッションにより、容量制限は事実上なくなりましたが、パフォーマンスを考慮して50,000件の上限を設定しています。
