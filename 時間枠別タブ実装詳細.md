# 時間枠別タブ機能 実装詳細

## 📋 実装概要

取引時間別（15秒・30秒・60秒・3分・5分）に切り替えられるタブ機能を実装しました。
各時間枠で最適化された分析設定を使用し、ユーザーが取引スタイルに合わせた分析を受けられます。

### 4. 時間枠別設定オブジェクト

**場所**: グローバル定義 (行33-104)

各時間枠ごとに最適化されたパラメータを定義:

```javascript
const TIMEFRAME_CONFIGS = {
  15: {
    label: '15秒',
    updateInterval: 10,      // 10秒ごとに分析更新
    dataWindow: 60,          // 直近60秒のデータを使用
    minDataPoints: 30,       // 最低30秒のデータが必要
    weights: {
      macd: 1.5,
      adx: 1.0,
      stochastic: 1.5,
      atr: 1.0,
      roc: 2.5,             // 超短期は勢い（ROC）を重視
      sentiment: 2.5        // センチメントも重要
    }
  },
  30: {
    label: '30秒',
    updateInterval: 15,      // 15秒ごとに更新
    dataWindow: 120,         // 直近120秒
    minDataPoints: 60,
    weights: {
      macd: 1.8,
      adx: 1.2,
      stochastic: 1.8,
      atr: 1.2,
      roc: 2.2,
      sentiment: 2.0
    }
  },
  60: {
    label: '60秒',
    updateInterval: 20,      // 20秒ごとに更新
    dataWindow: 180,         // 直近180秒
    minDataPoints: 90,
    weights: {
      macd: 2.0,
      adx: 1.5,
      stochastic: 2.0,
      atr: 1.5,
      roc: 2.0,
      sentiment: 1.8
    }
  },
  180: {
    label: '3分',
    updateInterval: 30,      // 30秒ごとに更新
    dataWindow: 300,         // 直近300秒（5分）
    minDataPoints: 120,
    weights: {
      macd: 2.3,            // 中期はMACDとADXを重視
      adx: 2.0,
      stochastic: 1.5,
      atr: 1.8,
      roc: 1.5,
      sentiment: 1.5
    }
  },
  300: {
    label: '5分',
    updateInterval: 60,      // 60秒ごとに更新
    dataWindow: 300,         // 全データ（300秒）
    minDataPoints: 150,
    weights: {
      macd: 2.5,            // 長期はMACDとADXを最重視
      adx: 2.5,
      stochastic: 1.2,
      atr: 2.0,
      roc: 1.0,
      sentiment: 1.0
    }
  }
};
```

**設計方針**:
- **短期（15秒・30秒）**: ROC（勢い）とセンチメントを重視
- **中期（60秒・3分）**: バランス型、MACDとADXの比重を増やす
- **長期（5分）**: MACD・ADXを最重視、トレンド追従型

### 5. タブ切り替え機能

**場所**: switchTimeframe関数 (行669-702)

**動作フロー**:
```
ユーザーがタブをクリック
  ↓
switchTimeframe(新しい時間枠)が呼ばれる
  ↓
currentTimeframe を更新
  ↓
タブのアクティブ状態を更新（CSSクラス切り替え）
  ↓
時間枠情報エリアを更新
  ↓
既存のpriceHistoryを使って即座に再分析
  ↓
UI更新（メインシグナル、全時間枠信号機、推奨アクション）
```

**実装コード**:
```javascript
function switchTimeframe(newTimeframe) {
  console.log(`[TheOption Analyzer] ⏱️ 時間枠切り替え: ${TIMEFRAME_CONFIGS[currentTimeframe].label} → ${TIMEFRAME_CONFIGS[newTimeframe].label}`);

  // 現在の時間枠を更新
  currentTimeframe = newTimeframe;

  // タブのアクティブ状態を更新
  document.querySelectorAll('.timeframe-tab').forEach(tab => {
    if (parseInt(tab.dataset.timeframe) === newTimeframe) {
      tab.classList.add('active');
    } else {
      tab.classList.remove('active');
    }
  });

  // 時間枠情報を更新
  updateTimeframeInfo();

  // データは引き継がれる - 即座に再分析
  const currentPrice = getCurrentPriceFromDOM();
  if (currentPrice && priceHistory.length > 0) {
    console.log(`[TheOption Analyzer] 🔄 ${TIMEFRAME_CONFIGS[newTimeframe].label}用の設定で再分析を実行...`);
    performAnalysis(currentPrice);
  } else {
    console.log(`[TheOption Analyzer] ⚠️ まだデータが不足しています`);
  }
}
```

**重要な仕組み**:
- **データ引き継ぎ**: `priceHistory`, `tickData`, `candles` はグローバル変数なので自動的に保持される
- **即座に再分析**: タブ切り替え直後に `performAnalysis()` を実行
- **設定だけ変更**: 同じデータに対して、異なる設定（重み、データ範囲）で分析

### 6. 更新頻度の動的調整

**場所**: 価格監視ループ (行798-805)

**動作**:
```javascript
// 1秒ごとに価格を取得
setInterval(() => {
  const price = getCurrentPriceFromDOM();
  if (!price) return;

  const now = Date.now();
  const config = TIMEFRAME_CONFIGS[currentTimeframe];
  const timeSinceLastAnalysis = (now - lastAnalysisTime) / 1000;

  // 時間枠の更新間隔に達したら分析実行
  if (timeSinceLastAnalysis >= config.updateInterval) {
    performAnalysis(price);
    lastAnalysisTime = now;
  }
}, 1000);
```

**例**:
- 15秒タブ選択中: 10秒ごとに分析
- 3分タブ選択中: 30秒ごとに分析
- 5分タブ選択中: 60秒ごとに分析

**メリット**:
- CPUリソースの節約
- 長期分析では頻繁な更新は不要
- ノイズの削減

### 7. 時間枠別分析実行

**場所**: performAnalysis関数 (行820-872)

**処理フロー**:
```
performAnalysis(currentPrice)
  ↓
現在の時間枠の設定を取得（TIMEFRAME_CONFIGS[currentTimeframe]）
  ↓
最低データ数をチェック
  ├─ 不足 → 「データ収集中」表示
  └─ 十分 → 分析実行
       ↓
時間枠に応じたデータ範囲を切り出し
  例: 15秒なら直近60秒、5分なら全300秒
       ↓
時間枠別の重み付けで多次元分析
  multiDimAnalyzer.analyzeTimeframe(data, currentTimeframe)
       ↓
ML予測を取得（通貨ペア別）
       ↓
結果をUIに反映
```

**実装コード**:
```javascript
function performAnalysis(currentPrice) {
  const config = TIMEFRAME_CONFIGS[currentTimeframe];

  // 最低データ数チェック
  if (priceHistory.length < config.minDataPoints) {
    updateUI({
      status: 'COLLECTING',
      message: `データ収集中... (${priceHistory.length}/${config.minDataPoints}秒)`,
      timeframe: currentTimeframe
    });
    return;
  }

  // 時間枠に応じたデータ範囲を取得
  const relevantPrices = priceHistory.slice(-config.dataWindow);
  const relevantTicks = tickData.slice(-config.dataWindow);

  console.log(`[TheOption Analyzer] 📊 ${config.label}用の分析を実行 (データ範囲: 直近${config.dataWindow}秒)`);

  // 多次元分析（時間枠別の重み付け）
  const multiDimResult = multiDimAnalyzer.analyzeTimeframe({
    prices: relevantPrices,
    candles: candles,
    ticks: relevantTicks
  }, currentTimeframe);

  // ML予測（通貨ペア別）
  const mlResult = mlSystem.getPredictions(currentAsset);

  // UI更新
  updateUI({
    status: 'ACTIVE',
    multiDim: multiDimResult,
    ml: mlResult,
    mlStats: mlSystem.getStats(currentAsset)
  });
}
```

**行997-1026**:
```javascript
function getCurrentTimeframeSignal(multiDim, ml) {
  // 多次元分析結果
  let signal = multiDim.signal;
  let conf = multiDim.confidence;

  // ML予測があればそちらを優先
  if (ml.status === 'READY' && ml.predictions[`${currentTimeframe}s`]) {
    const mlPred = ml.predictions[`${currentTimeframe}s`];
    if (mlPred.confidence > conf) {
      signal = mlPred.prediction;
      conf = mlPred.confidence;
    }
  }

  let direction;
  if (signal === 'HIGH') {
    direction = 'HIGH';
  } else if (signal === 'LOW') {
    direction = 'LOW';
  } else {
    direction = '見送り';
  }

  return {
    signal,
    confidence: conf,
    direction,
    timeframe: TIMEFRAME_CONFIGS[currentTimeframe].label
  };
}
```

```javascript
function updateSignalLights(multiDim, ml) {
  const timeframes = [
    { id: 'signal-15s', seconds: 15 },
    { id: 'signal-30s', seconds: 30 },
    { id: 'signal-60s', seconds: 60 },
    { id: 'signal-3m', seconds: 180 },
    { id: 'signal-5m', seconds: 300 }
  ];

  timeframes.forEach(tf => {
    // 各時間枠ごとに独立した分析を実行
    const timeframeAnalysis = multiDimAnalyzer.analyzeTimeframe({
      prices: priceHistory,
      candles: candles,
      ticks: tickData
    }, tf.seconds);

    let signal = timeframeAnalysis.signal;
    let conf = timeframeAnalysis.confidence;

    // ML予測があればそちらを優先
    if (ml.status === 'READY' && ml.predictions[`${tf.seconds}s`]) {
      const mlPred = ml.predictions[`${tf.seconds}s`];
      if (mlPred.confidence > conf) {
        signal = mlPred.prediction;
        conf = mlPred.confidence;
      }
    }

    // 信号機の色を更新
    const element = document.getElementById(tf.id);
    const light = element.querySelector('.signal-light');
    const confidence = element.querySelector('.signal-confidence');

    if (conf >= 70) {
      if (signal === 'HIGH') {
        light.textContent = '🟢';
      } else if (signal === 'LOW') {
        light.textContent = '🔴';
      } else {
        light.textContent = '⚪';
      }
    } else {
      light.textContent = '⚪';
    }

    confidence.textContent = `${conf}%`;
  });
}
```

## 🔧 内部ロジック

### 重み付けの設計思想

#### 短期取引（15秒・30秒）
- **ROC（勢い）を最重視**: 短期的な価格変動率が重要
- **Sentiment（センチメント）も重要**: 市場の瞬間的な心理
- **MACD・ADXは控えめ**: 長期トレンドは参考程度

#### 中期取引（60秒・3分）
- **バランス型**: すべての指標を考慮
- **MACDの比重を増やす**: トレンドの方向性を確認

#### 長期取引（5分）
- **MACD・ADX最重視**: 明確なトレンドを追従
- **ATRも重要**: ボラティリティの評価
- **ROC・Sentimentは控えめ**: ノイズとして扱う

### 更新頻度の設計思想

- **短期**: 頻繁に更新（10-15秒）してタイミングを逃さない
- **中期**: 適度な更新（20-30秒）でノイズを削減
- **長期**: 低頻度（60秒）で安定したシグナル

### 推奨の使い方

- **15秒取引**: 超短期、勢いに乗る
  - 🟢 HIGH（70%以上）→ HIGHでエントリー
  - 🔴 LOW（70%以上）→ LOWでエントリー

- **30秒-60秒取引**: 短期、バランス型
  - 信頼度75%以上を推奨

- **3分-5分取引**: 中長期、トレンド追従
  - 信頼度80%以上を推奨
  - 全時間枠の信号機が一致している場合は特に有利
