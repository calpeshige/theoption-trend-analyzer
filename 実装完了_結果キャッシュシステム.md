# 実装完了：結果キャッシュシステム

## 📋 解決した問題

### 問題1: タブ切り替え時に前の取引時間の結果が引き継がれる

**ユーザーからの報告**:
> 現状を時間取引時間のタブを変更しても前の取引時間の結果が見送り30%のように引き継がれてしまいます。そのため混乱が起きます。

**原因**:
- タブ切り替え時に各時間枠の分析結果を保存していなかった
- `currentTimeframe`の分析結果だけがメモリに残っていた
- タブを切り替えても、前回の時間枠の結果がそのまま表示されていた

### 問題2: メイン表示と個別時間枠の信頼度がずれる

**ユーザーからの報告**:
> 大きく表示されているパーセンテージの結果とその下に表示されている各取引時間での結果がずれています。例えば15秒取引のタブを選択し、パーセンテージが38%で出ているのにその下の15秒のパーセンテージは34%のようにずれています。

**原因**:
```javascript
// メイン表示
getCurrentTimeframeSignal(multiDim) {
  return multiDim.confidence;  // performAnalysis()の結果を使用
}

// 小さいシグナル（問題の箇所）
updateSignalLights(multiDim, ml) {
  timeframes.forEach(tf => {
    // 毎回再計算していた！
    const analysis = multiDimAnalyzer.analyzeTimeframe(..., tf.seconds);
    conf = analysis.confidence;  // 異なる結果になる
  });
}
```

**なぜずれるのか**:
1. `performAnalysis()`は10秒～60秒おきに1回だけ実行
2. `updateSignalLights()`はUI更新のたびに全時間枠を再計算
3. 実行タイミングが異なるため、微妙にデータが違う
4. 結果として、同じ15秒でも38%と34%のように差が出る

### 3. switchTimeframe の修正

**変更箇所** (`theoption-analyzer.js` 701-718行目):

タブ切り替え時にキャッシュから結果を復元：

```javascript
// キャッシュされた分析結果があれば表示
const cachedResult = timeframeResults[newTimeframe];
if (cachedResult) {
  console.log(`[TheOption Analyzer] 📋 ${TIMEFRAME_CONFIGS[newTimeframe].label}の前回分析結果を表示します`);
  updateUI({
    status: 'ACTIVE',
    multiDim: cachedResult.multiDim,
    ml: cachedResult.ml,
    mlStats: cachedResult.mlStats
  });
} else {
  console.log(`[TheOption Analyzer] ⏳ ${TIMEFRAME_CONFIGS[newTimeframe].label}の分析結果待ち...次回の更新で分析します`);
  updateUI({
    status: 'COLLECTING',
    message: `${TIMEFRAME_CONFIGS[newTimeframe].label} - 分析待ち...`,
    timeframe: newTimeframe
  });
}
```

**動作例**:

```
初期状態:
timeframeResults = { 15: null, 30: null, ... }

10秒後: 15秒の分析完了
timeframeResults = { 15: {HIGH 65%}, 30: null, ... }
表示: HIGH 65%

ユーザーが30秒タブに切り替え
→ timeframeResults[30] = null なので「分析待ち...」表示

25秒後: 30秒の分析完了
timeframeResults = { 15: {HIGH 65%}, 30: {LOW 58%}, ... }
表示: LOW 58%

ユーザーが15秒タブに戻す
→ timeframeResults[15] = {HIGH 65%} を表示
→ 前回の結果が正しく復元される！
```

### 4. updateSignalLights の修正

**変更箇所** (`theoption-analyzer.js` 1107-1127行目):

小さいシグナルランプも再計算せず、キャッシュを使用：

```javascript
// キャッシュから結果を取得（再計算しない）
const cachedResult = timeframeResults[tf.seconds];

if (cachedResult) {
  // キャッシュがあればそれを使用
  signal = cachedResult.multiDim.signal;
  conf = cachedResult.multiDim.confidence;

  // ML予測があればそちらを優先
  if (cachedResult.ml.status === 'READY' && cachedResult.ml.predictions[`${tf.seconds}s`]) {
    const mlPred = cachedResult.ml.predictions[`${tf.seconds}s`];
    if (mlPred.confidence > conf) {
      signal = mlPred.prediction;
      conf = mlPred.confidence;
    }
  }
} else {
  // キャッシュがない場合は「分析待ち」表示
  signal = 'WAIT';
  conf = 0;
}
```

**Before（問題あり）**:
```javascript
// 毎回再計算
const timeframeAnalysis = multiDimAnalyzer.analyzeTimeframe(..., tf.seconds);

// メイン表示: 38%
// 小シグナル: 34% ← 再計算で微妙に違う！
```

**After（修正後）**:
```javascript
// キャッシュから取得
const cachedResult = timeframeResults[tf.seconds];
conf = cachedResult.multiDim.confidence;

// メイン表示: 38%
// 小シグナル: 38% ← 同じキャッシュを使うので一致！
```

## 🔍 技術的な詳細

### キャッシュの更新タイミング

```javascript
// 各時間枠の更新頻度
15秒取引: 10秒ごと  → timeframeResults[15]を10秒ごとに更新
30秒取引: 15秒ごと  → timeframeResults[30]を15秒ごとに更新
60秒取引: 20秒ごと  → timeframeResults[60]を20秒ごとに更新
3分取引:  30秒ごと  → timeframeResults[180]を30秒ごとに更新
5分取引:  60秒ごと  → timeframeResults[300]を60秒ごとに更新
```

### メモリ管理

**キャッシュサイズ**:
- 各時間枠: 約5-10KB
- 全5時間枠: 約25-50KB
- メモリ負荷: 非常に軽い

**データの鮮度**:
- `timestamp`フィールドで最終更新時刻を記録
- 必要に応じて古いキャッシュの判定が可能（現在は未実装）
