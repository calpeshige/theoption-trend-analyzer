# デバッグ: 価格検出の詳細ログ追加

## 問題

価格が画面上では動いているのに、拡張機能では常に95.000という固定値しか検出されない。

## 原因の仮説

`getCurrentPriceFromDOM()` 関数が、リアルタイムで更新される価格要素ではなく、静的な値を持つ別の要素を選択している可能性が高い。

## 実施した修正

### [theoption-analyzer.js:3170-3211](theoption-analyzer.js#L3170-L3211)

価格検出関数に詳細なデバッグログを追加しました:

```javascript
function getCurrentPriceFromDOM() {
  const selectors = [
    '.rate',
    '[class*="rate"]',
    '[class*="price"]',
    '[class*="Rate"]',
    '[class*="Price"]',
    '.current-rate',
    '.asset-rate',
    '[data-rate]'
  ];

  for (const selector of selectors) {
    const elements = document.querySelectorAll(selector);
    for (const el of elements) {
      const text = el.textContent.trim();
      const number = parseFloat(text.replace(/[^0-9.]/g, ''));

      if (!isNaN(number) && number > 0 && number < 1000000) {
        // 詳細ログ: どの要素から価格を取得しているか特定
        console.log(`[TheOption Analyzer] 💹 価格検出: ${number}`);
        console.log(`[TheOption Analyzer] 🔍 セレクタ: ${selector}`);
        console.log(`[TheOption Analyzer] 🔍 要素情報:`, {
          tagName: el.tagName,
          className: el.className,
          id: el.id,
          textContent: text,
          innerHTML: el.innerHTML.substring(0, 100)
        });
        console.log(`[TheOption Analyzer] 🔍 親要素:`, {
          tagName: el.parentElement?.tagName,
          className: el.parentElement?.className,
          id: el.parentElement?.id
        });
        return number;
      }
    }
  }

  console.warn('[TheOption Analyzer] ⚠️ 価格要素が見つかりません');
  return null;
}
```

**変更点:**

1. **`priceDetectionLogged` フラグを削除**: 毎回ログを出力するようにした（以前は初回のみ）
2. **要素の詳細情報を出力**:
   - `tagName`: HTML要素の種類（DIV, SPANなど）
   - `className`: クラス名（どのクラスを持っているか）
   - `id`: ID属性
   - `textContent`: 要素のテキスト内容
   - `innerHTML`: 要素の内部HTML（最初の100文字）
3. **親要素の情報も出力**: 要素の階層構造を理解するため

## 期待される結果

拡張機能をリロードしてTheOptionページを開くと、以下のような詳細ログが表示されます:

```
[TheOption Analyzer] 💹 価格検出: 95
[TheOption Analyzer] 🔍 セレクタ: [class*="Rate"]
[TheOption Analyzer] 🔍 要素情報: {
  tagName: "DIV",
  className: "someRateClass",
  id: "",
  textContent: "95",
  innerHTML: "95"
}
[TheOption Analyzer] 🔍 親要素: {
  tagName: "DIV",
  className: "parentClass",
  id: "someId"
}
```

このログから、**どの要素が選択されているか**が明確になります。

## 次のステップ

このログを確認することで:

1. **静的な要素を選択している場合**
   - 例: ラベル、説明文、過去の価格など
   - → 正しいセレクタに変更する必要がある

2. **正しい要素だが更新されない場合**
   - TheOptionのサイトがReactなどで仮想DOMを使用している可能性
   - → MutationObserverなど別の方法で価格変更を検知する必要がある

3. **複数の要素がマッチする場合**
   - 最初に見つかった要素が静的なもの
   - → より具体的なセレクタで、リアルタイム価格要素のみを選択する

## テスト手順

1. **拡張機能をリロード**
   - `chrome://extensions/` で「再読み込み」をクリック

2. **TheOptionのトレーディングページをリロード**

3. **コンソールを開く**
   - F12キーを押す

4. **ログを確認**
   - `[TheOption Analyzer] 💹 価格検出:` のログを探す
   - `🔍 要素情報:` のオブジェクトを展開して詳細を確認

5. **情報を送ってください**:
   ```
   🔍 要素情報: {
     tagName: "?",
     className: "?",
     id: "?",
     textContent: "?",
     innerHTML: "?"
   }
   🔍 親要素: {
     tagName: "?",
     className: "?",
     id: "?"
   }
   ```

この情報があれば、正しいセレクタを特定できます。

## 補足: 価格検出の頻度

- 価格検出は **1秒ごと** に実行されます（line 2001: `}, 1000)`）
- 検出された価格は `priceHistory` 配列に追加されます（line 1946）
- `priceHistory` は最大300件まで保持されます（line 1947）

したがって、もし価格が実際に変動している場合、`最新5件の価格` ログに異なる値が表示されるはずです。
