## 実装概要

相場の動きが小さいデータを自動的にフィルタリングし、重要なデータだけを保存することで、以下を実現：
- **データ量の削減**: 動きのない相場データを保存しない
- **パフォーマンス向上**: 不要なデータがないため分析が高速化
- **予測精度の維持**: 重要なデータだけを使用するため精度は維持

```javascript
calculateDynamicThreshold(timeframe, atrPercent) {
  const config = this.segmentConfigs[timeframe];

  if (atrPercent && atrPercent > 0) {
    const atrBasedThreshold = atrPercent * config.atrMultiplier;
    threshold = Math.max(config.minThreshold, atrBasedThreshold);
  }

  return threshold;
}
```

## 2. データ保存フィルター

### フィルタリングロジック

**条件**: どれか1つの時間枠で動きがあるセグメントがあればデータ保存

```
15秒セグメント: FLAT, FLAT, UP, FLAT, FLAT, FLAT
                        ↑ 動きあり！
→ データ保存 ✅

全時間枠: 全セグメントがFLAT
→ データ保存しない ❌
```

### 実装詳細

**[machine-learning-system.js:129-138](machine-learning-system.js#L129-L138)**
```javascript
// データ保存フィルター
const shouldSaveData = this.hasMarketMovement(situation);
if (!shouldSaveData) {
  console.log('[ML] 📉 相場変動が閾値以下のためデータ保存をスキップ');
  return null;
}

console.log('[ML] ✅ 相場変動検出 - データを保存');
this.trainingData.push(situation);
```

**[machine-learning-system.js:213-237](machine-learning-system.js#L213-L237)**
```javascript
hasMarketMovement(situation) {
  const timeframes = [15, 30, 60, 180, 300];

  for (const tf of timeframes) {
    const segments = situation[`priceSegments${tf}s`];

    if (!segments || !segments.segments) continue;

    for (const segment of segments.segments) {
      // UPまたはDOWNが1つでもあれば保存
      if (segment.direction === 'UP' || segment.direction === 'DOWN') {
        return true;
      }
    }
  }

  // 全セグメントがFLAT
  return false;
}
```

## 3. 期待される効果

```

```

## 5. 調整方法

### 閾値を変更したい場合

[detailed-segment-analyzer.js:25-61](detailed-segment-analyzer.js#L25-L61) で調整：

```javascript
15: {
  atrMultiplier: 0.09,  // ← この値を変更
  minThreshold: 0.0032  // ← または最小値を変更
}
```

**より厳しく（FLAT増加）**: 係数を上げる（例: 0.09 → 0.15）
**より緩く（FLAT減少）**: 係数を下げる（例: 0.09 → 0.05）

### データ保存基準を変更したい場合

## 6. トラブルシューティング

### FLAT表示が出ない場合

**原因**: 閾値が低すぎる
**解決**: `atrMultiplier` を上げる

### データが全く保存されない場合

**原因**: 閾値が高すぎる
**解決**: `atrMultiplier` を下げる

### パフォーマンスがまだ重い場合
