# 類似度スコア詳細分析ログ追加

## 実施した変更

98%という高い類似度が多発する原因を特定するため、上位5件の類似パターンについて詳細なスコア内訳を表示するログを追加しました。

### 追加したログ機能

#### 1. 各指標のスコアを個別に記録 (`machine-learning-system.js:541-574`)

```javascript
// RSI時系列類似度（重み: 15点）
const rsiScore = this.compareTechIndicator(currentTech.rsi, pastTech.rsi, 15);
score += rsiScore;
maxScore += 15;
debugInfo.details.rsi = { score: rsiScore.toFixed(1), max: 15 };

// MACD時系列類似度（重み: 15点）
const macdScore = this.compareTechIndicator(currentTech.macd, pastTech.macd, 15);
score += macdScore;
maxScore += 15;
debugInfo.details.macd = { score: macdScore.toFixed(1), max: 15 };

// ... (他の指標も同様)
```

**目的**: 各テクニカル指標が何点獲得しているかを追跡

## 確認すべきポイント

### 1. NEUTRAL同士の高得点問題

もし98%の類似度パターンのほとんどが:
- `トレンド一致: 15/15 (現在=NEUTRAL, 過去=NEUTRAL)` または
- `トレンド一致: 7/15 (現在=NEUTRAL, 過去=UP)` のようにNEUTRALが関与

している場合、NEUTRAL判定が過剰にスコアを与えている可能性があります。

### 2. テクニカル指標の時系列比較の問題

もし `テクニカル指標スコア: 55.0+/60点` のように常に高得点の場合:
- `compareTechIndicator()` メソッドが NEUTRAL トレンド同士に高得点を与えている
- 時系列データの変化量が小さすぎて、ほとんどのパターンが似ていると判定される

可能性があります。

### 3. 期待される結果

正常な分析であれば:
- 98%の類似度: トレンド方向が完全一致 + 全指標が非常に近い値
- 70-80%の類似度: 一部の指標は一致するが、他は異なる
- 50-60%の類似度: 大まかな傾向のみ一致
