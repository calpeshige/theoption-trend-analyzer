# テクニカル分析の仕組み - 初心者向け完全解説

## 🎯 全体の流れ（5ステップ）

```
1. 価格データを集める（120秒分）
   ↓
2. 6つの指標で分析する
   ↓
3. 各指標が「上がる」「下がる」の強さを点数化（-10～+10点）
   ↓
4. 点数を合計して正規化（-100～+100点）
   ↓
5. 信号と信頼度％を決定
```

---

## 📊 ステップ1: 価格データを集める

### 何をしているか？

TheOptionの画面から、**1秒ごとに価格を記録**しています。

```
例：EUR/USD（ユーロ/米ドル）の価格
12:00:00 → 1.08501
12:00:01 → 1.08503
12:00:02 → 1.08505
...
12:02:00 → 1.08520（120秒分のデータ）
```

### なぜ120秒分？

- **15秒取引**には120秒分のデータが必要
- MACD（後述）という指標が最低26個のデータを必要とする
- データが少ないと正確な分析ができない

---

## 🔍 ステップ2: 6つの指標で分析する

### 指標とは？

価格の動きを**数値化して判断しやすくする道具**です。

例えば：
- 体温計 → 体調を数値化
- 血圧計 → 健康状態を数値化
- **テクニカル指標** → 相場の状態を数値化

---

## 📈 6つの指標の詳細解説

### 1. MACD（マックディー）- 重み20%

**何を見ているか？**
短期の価格平均と長期の価格平均の差を見て、**勢いの変化**を判断します。

**例え話：電車の速度**
```
短期平均（12秒平均） = 時速50km
長期平均（26秒平均） = 時速40km
差（MACD） = +10km → 加速している！
```

**計算方法**:
```javascript
// 1. 短期平均（12秒）と長期平均（26秒）を計算
短期平均 = 直近12秒の価格の平均
長期平均 = 直近26秒の価格の平均

// 2. 差を取る
MACD = 短期平均 - 長期平均

// 3. シグナルライン（MACD の80%）
Signal = MACD × 0.8

// 4. ヒストグラム（差）
Histogram = MACD - Signal

// 5. 強度スコア（-10 ～ +10）
Strength = Histogram × 100 × 感度係数
```

**15秒取引の感度調整**:
```
通常: Histogram × 100 → 0.0001 × 100 = 0.01点
15秒: Histogram × 100 × 100 = 0.0001 × 10000 = 1点
```

**判定**:
- プラス → 上昇トレンド（HIGHを推奨）
- マイナス → 下降トレンド（LOWを推奨）

---

### 2. ADX（エーディーエックス）- 重み15%

**何を見ているか？**
**トレンドの強さ**を判断します（上昇・下降は問わず）。

**例え話：風の強さ**
```
ADX = 10 → そよ風（弱いトレンド）
ADX = 25 → 強風（明確なトレンド）
ADX = 50 → 暴風（非常に強いトレンド）
```

**計算方法**:
```javascript
// 1. 上昇幅と下降幅を計算
plusDM = 今日の高値 - 昨日の高値（プラスの場合のみ）
minusDM = 昨日の安値 - 今日の安値（プラスの場合のみ）

// 2. True Range（価格の変動幅）
TR = max(高値-安値, |高値-前回終値|, |安値-前回終値|)

// 3. 方向性指標
plusDI = (plusDM / TR) × 100
minusDI = (minusDM / TR) × 100

// 4. ADX（方向性の強さ）
ADX = |plusDI - minusDI| / (plusDI + minusDI) × 100

// 5. 強度スコア（0 ～ 10）
Strength = ADX / 5
```

**判定**:
- ADX > 25 → 強いトレンド（信頼度アップ）
- ADX < 20 → 弱いトレンド（信頼度ダウン）

---

### 3. Stochastic（ストキャスティクス）- 重み15%

**何を見ているか？**
**買われすぎ・売られすぎ**を判断します。

**例え話：ゴムの伸び**
```
価格 = ゴムの長さ

K値 = 80 → 限界まで伸びた（買われすぎ→下がる）
K値 = 20 → 限界まで縮んだ（売られすぎ→上がる）
K値 = 50 → 中立
```

**計算方法**:
```javascript
// 1. 直近14秒の最高値と最安値を取得
最高値 = 直近14秒で一番高い価格
最安値 = 直近14秒で一番安い価格

// 2. K値（現在位置のパーセント）
K = (現在価格 - 最安値) / (最高値 - 最安値) × 100

// 3. D値（K値の移動平均、簡易版）
D = K × 0.9

// 4. 判定
if (K > 80 && D > 80) {
  Signal = '買われすぎ';
  Strength = -5; // 下がる方向
} else if (K < 20 && D < 20) {
  Signal = '売られすぎ';
  Strength = -5; // 実装上は反転シグナル
} else if (K > D && K < 80) {
  Signal = '上昇中';
  Strength = 5;
} else if (K < D && K > 20) {
  Signal = '下降中';
  Strength = -5;
}
```

**判定**:
- K > 80 → 買われすぎ（そろそろ下がる）
- K < 20 → 売られすぎ（そろそろ上がる）
- K > D → 上昇トレンド
- K < D → 下降トレンド

---

### 4. ATR（エーティーアール）- 重み10%

**何を見ているか？**
**価格の変動の激しさ（ボラティリティ）**を判断します。

**例え話：波の高さ**
```
ATR = 小 → さざ波（動きが少ない）
ATR = 中 → 中くらいの波（適度な変動）
ATR = 大 → 大波（激しい変動）
```

**計算方法**:
```javascript
// 1. True Range（1秒間の価格変動幅）
TR = max(
  高値 - 安値,
  |高値 - 前回終値|,
  |安値 - 前回終値|
)

// 2. ATR（平均True Range）
ATR = 直近14秒のTRの平均

// 3. ATR%（価格に対する割合）
ATR% = (ATR / 現在価格) × 100

// 4. 判定（15秒取引用の閾値）
if (ATR% > 0.005%) {
  Volatility = 'HIGH';
  Strength = 3;
} else if (ATR% > 0.002%) {
  Volatility = 'MODERATE';
  Strength = 2;
} else {
  Volatility = 'LOW';
  Strength = 1;
}
```

**判定**:
- 高ボラティリティ → 大きく動く可能性（信号の信頼度アップ）
- 低ボラティリティ → あまり動かない（信号の信頼度ダウン）

---

### 5. ROC（アールオーシー）- 重み20%

**何を見ているか？**
**一定期間でどれだけ価格が変化したか**を％で表します。

**例え話：成長率**
```
10秒前の価格 = 1.08500
現在の価格   = 1.08550
ROC = (1.08550 - 1.08500) / 1.08500 × 100 = 0.046%
```

**計算方法**:
```javascript
// 1. ROC計算
現在価格 = 1.08550
10秒前の価格 = 1.08500
ROC = ((現在価格 - 10秒前の価格) / 10秒前の価格) × 100

// 2. 判定（15秒取引用の閾値）
strongThreshold = 0.01%  // 1.0% / 100
weakThreshold = 0.003%   // 0.3% / 100

if (ROC > 0.01%) {
  Signal = 'STRONG_UP';
  Strength = 10;
} else if (ROC > 0.003%) {
  Signal = 'UP';
  Strength = 5;
} else if (ROC < -0.01%) {
  Signal = 'STRONG_DOWN';
  Strength = -10;
} else if (ROC < -0.003%) {
  Signal = 'DOWN';
  Strength = -5;
} else {
  Signal = 'NEUTRAL';
  Strength = 0;
}
```

**判定**:
- ROC > 0.01% → 強い上昇
- ROC > 0.003% → 上昇
- ROC < -0.01% → 強い下降
- ROC < -0.003% → 下降

---

### 6. Market Sentiment（マーケットセンチメント）- 重み20%

**何を見ているか？**
**上がったティックと下がったティックの比率**から市場の雰囲気を判断します。

**例え話：投票結果**
```
直近60ティック:
上昇 = 40回
下降 = 20回
上昇比率 = 40/60 = 66.7%

判定 → 強気（BULLISH）
```

**計算方法**:
```javascript
// 1. 直近60ティックを取得
recent = 直近60個のティックデータ

// 2. 上昇ティック比率
upTicks = 価格が上がったティックの数
upRatio = upTicks / 60

// 3. 平均変化幅
avgChange = 各ティックの変化幅の絶対値の平均

// 4. センチメント判定
if (upRatio > 0.65) {
  Sentiment = 'BULLISH'; // 強気
  Strength = 5;
} else if (upRatio < 0.35) {
  Sentiment = 'BEARISH'; // 弱気
  Strength = -5;
} else {
  Sentiment = 'NEUTRAL';
  Strength = 0;
}

// 5. 強度調整（15秒取引用の閾値）
highThreshold = 0.1    // 10 / 100
moderateThreshold = 0.05  // 5 / 100

if (avgChange > 0.1) {
  Intensity = 'HIGH';
  Strength *= 1.5; // 強度1.5倍
} else if (avgChange > 0.05) {
  Intensity = 'MODERATE';
  // そのまま
} else {
  Intensity = 'LOW';
  Strength *= 0.5; // 強度半減
}
```

**判定**:
- upRatio > 65% → 強気相場（HIGHを推奨）
- upRatio < 35% → 弱気相場（LOWを推奨）
- 変化幅が大きい → 強度アップ

## 🧮 ステップ3: 点数を合計する

### 各指標の点数に重みをかける

```javascript
totalScore = 0;
maxScore = 100;

// MACD（重み: 20%）
totalScore += macd.strength × 2;  // -10～+10 → -20～+20
maxScore = 20;

// Stochastic（重み: 15%）
totalScore += stochastic.strength × 1.5;  // -5～+5 → -7.5～+7.5
maxScore += 15;

// ATR（重み: 10%）- ボラティリティ加算（方向性なし）
volatilityBonus = atr.strength;  // 1～3
maxScore += 10;

// ROC（重み: 20%）
totalScore += roc.strength × 2;  // -10～+10 → -20～+20
maxScore += 20;

// Sentiment（重み: 20%）
totalScore += sentiment.strength × 2;  // -7.5～+7.5 → -15～+15
maxScore += 20;

// ADX（重み: 15%）- トレンド強度（方向性なし）
trendConfidence = adx.strength × 1.5;  // 0～10 → 0～15
maxScore += 15;
```

### 実際の計算例（15秒取引）

```javascript
// 各指標の強度
macd.strength = 2.0      // 上昇
stochastic.strength = 0  // 中立
atr.strength = 3         // 高ボラティリティ
roc.strength = 10        // 強い上昇
sentiment.strength = 7.5 // 強気

// 合計スコア計算
totalScore = 2.0×2 + 0×1.5 + 10×2 + 7.5×2
           = 4 + 0 + 20 + 15
           = 39

maxScore = 100
```

## 📐 ステップ4: 正規化（-100～+100に変換）

### なぜ正規化するのか？

各指標の点数の範囲がバラバラなので、**統一した基準**で判断するため。

```javascript
normalizedScore = (totalScore / maxScore) × 100
                = (39 / 100) × 100
                = 39
```

**意味**:
- +100 = 最強の上昇信号
- 0 = 中立
- -100 = 最強の下降信号
- **39 = やや強い上昇信号**

## 🎯 ステップ5: 信号と信頼度％を決定

### 信号の判定

```javascript
if (normalizedScore > 50 && trendConfidence > 7) {
  // 非常に強い上昇
  signal = 'STRONG_HIGH';
  confidence = Math.min(95, 70 + normalizedScore / 3);

} else if (normalizedScore > 15) {
  // 上昇
  signal = 'HIGH';
  confidence = Math.min(85, 60 + normalizedScore / 4);

} else if (normalizedScore < -50 && trendConfidence > 7) {
  // 非常に強い下降
  signal = 'STRONG_LOW';
  confidence = Math.min(95, 70 + Math.abs(normalizedScore) / 3);

} else if (normalizedScore < -15) {
  // 下降
  signal = 'LOW';
  confidence = Math.min(85, 60 + Math.abs(normalizedScore) / 4);

} else {
  // 中立
  signal = 'NEUTRAL';
  confidence = 50 + Math.abs(normalizedScore) × 1.5;
}
```

### 実際の例で計算

**ケース1: normalizedScore = 39**

```javascript
// 判定
39 > 15 → signal = 'HIGH'

// 信頼度計算
confidence = 60 + 39 / 4
           = 60 + 9.75
           = 69.75
           = 70%（四捨五入）
```

**結果**: HIGH推奨 70%

---

**ケース2: normalizedScore = 60, trendConfidence = 10**

```javascript
// 判定
60 > 50 && 10 > 7 → signal = 'STRONG_HIGH'

// 信頼度計算
confidence = 70 + 60 / 3
           = 70 + 20
           = 90%
```

**結果**: STRONG_HIGH 90%

---

**ケース3: normalizedScore = 5**

```javascript
// 判定
-15 < 5 < 15 → signal = 'NEUTRAL'

// 信頼度計算
confidence = 50 + |5| × 1.5
           = 50 + 7.5
           = 57.5
           = 58%（四捨五入）
```

**結果**: 見送り 58%

## 📊 信頼度％の意味

### 信頼度の範囲

| 信号 | normalizedScore | 信頼度範囲 | 意味 |
|------|----------------|-----------|------|
| STRONG_HIGH | > 50（強トレンド） | 70～95% | 非常に強い上昇信号 |
| HIGH | 15～50 | 60～85% | 上昇信号 |
| NEUTRAL | -15～15 | 50～72% | 見送り推奨 |
| LOW | -50～-15 | 60～85% | 下降信号 |
| STRONG_LOW | < -50（強トレンド） | 70～95% | 非常に強い下降信号 |

### 信頼度の計算式

**HIGH/LOW の場合**:
```
信頼度 = 60 + |normalizedScore| / 4
```

例：
- normalizedScore = 20 → confidence = 60 + 20/4 = 65%
- normalizedScore = 40 → confidence = 60 + 40/4 = 70%
- normalizedScore = 80 → confidence = 60 + 80/4 = 80%

**STRONG_HIGH/STRONG_LOW の場合**:
```
信頼度 = 70 + |normalizedScore| / 3
```

例：
- normalizedScore = 60 → confidence = 70 + 60/3 = 90%
- normalizedScore = 75 → confidence = 70 + 75/3 = 95%

**NEUTRAL の場合**:
```
信頼度 = 50 + |normalizedScore| × 1.5
```

例：
- normalizedScore = 0 → confidence = 50%
- normalizedScore = 10 → confidence = 50 + 10×1.5 = 65%
- normalizedScore = 15 → confidence = 50 + 15×1.5 = 72.5%

## 🔧 15秒取引専用の感度調整

### なぜ感度調整が必要？

15秒取引は価格変動が**非常に小さい**（0.001～0.01%程度）ため、通常の閾値では全ての指標が「0点」になってしまいます。

### 感度係数

```javascript
時間枠 → 感度係数
15秒  → 100倍
30秒  → 50倍
60秒  → 20倍
3分   → 5倍
5分   → 1倍（通常）
```

### 各指標の調整

**MACD**:
```
通常: histogram × 100
15秒: histogram × 100 × 100 = histogram × 10000
```

**ROC**:
```
通常: 1.0%以上で強い上昇
15秒: 0.01%以上で強い上昇（1/100）
```

**ATR**:
```
通常: 0.5%以上でHIGHボラティリティ
15秒: 0.005%以上でHIGHボラティリティ（1/100）
```

**Sentiment**:
```
通常: avgChange > 10 で強度アップ
15秒: avgChange > 0.1 で強度アップ（1/100）
```

これにより、超短期の小さな変動でも適切に検出できるようになります。

## 🎉 まとめ

### テクニカル分析の流れ

1. **データ収集**: 120秒分の価格を記録
2. **6つの指標**: MACD、ADX、Stochastic、ATR、ROC、Sentiment
3. **点数化**: 各指標が -10～+10点の強度を返す
4. **重み付け合計**: 合計100点満点で計算
5. **正規化**: -100～+100点に変換
6. **判定**: スコアに応じて信号と信頼度％を決定

### 信頼度％の決まり方

- **60%以上**: 推奨（HIGH または LOW）
- **70%以上**: 強い推奨
- **85%以上**: 非常に強い推奨（最大値）
- **50～60%**: 見送り推奨

### ポイント

- ✅ 6つの指標を総合的に判断（1つだけではない）
- ✅ 各指標に重み付けがある（ROC、Sentiment、MACDが重い）
- ✅ 15秒取引専用に感度を100倍に調整
- ✅ 信頼度％はnormalizedScoreから計算される

**これが TheOption Analyzer のテクニカル分析システムです！**
