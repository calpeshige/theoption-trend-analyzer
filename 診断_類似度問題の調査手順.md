# 類似度95%多発問題 - 診断手順

## 現状

ユーザーから報告された問題:
- **214件の学習データで160〜198件が類似パターンと判定される (75-92%)**
- 類似度閾値70%でも、ほとんどのデータが「類似」と判定される
- 期待値: 70%閾値なら5-15%程度の一致率

## 既に実施済みの修正

[修正完了_類似度計算とストレージ容量.md](修正完了_類似度計算とストレージ容量.md) で以下の修正を実施:

### 修正1: NEUTRAL同士のスコアリングを大幅減点

**テクニカル指標 (machine-learning-system.js:814-823)**
- NEUTRAL⇄NEUTRAL: **40% → 5%** (各指標の重みに対して)
- NEUTRAL⇄UP/DOWN: **20% → 0%**

**価格パターン (machine-learning-system.js:621-633)**
- NEUTRAL⇄NEUTRAL: **15点 → 3点**
- NEUTRAL⇄UP/DOWN: **7点 → 0点**

### 修正2: ストレージ容量問題

- データ保存上限: **50,000件 → 3,000件**
- メモリ内データ上限: **60,000件 → 3,000件**

## 問題が継続している可能性の原因

### 原因1: 拡張機能が再読み込みされていない ⚠️

**確認方法:**
1. Chrome拡張機能管理画面 (`chrome://extensions/`)
2. 「TheOption Trend Analyzer」の「再読み込み」ボタンをクリック
3. TheOptionのページもリロード

### 原因2: データがNEUTRAL以外で高スコアを獲得している

修正によってNEUTRALスコアは下がったが、以下の要素で高得点を獲得している可能性:
- 傾きが類似: `Δ0.000〜0.003` で10点満点
- 上昇率が類似: `Δ0.01` で7点満点
- ボラティリティが類似: `Δ0.01` で5点満点
- パターンタイプ一致: 3点満点

低ボラティリティ相場では、これらが全て満点近くを取り、NEUTRAL減点後でも高スコアになる可能性がある。

### 原因3: テクニカル指標の velocity, volatility, range が類似

NEUTRAL同士は減点されるが、velocity(変化速度)、volatility(ボラティリティ)、range(変動幅)の類似度で30%+20%+10%=60%のスコアを獲得できる。

低ボラティリティ相場では、全てのデータで velocity≈0, volatility≈小, range≈小 となり、高得点になる。

## 診断手順

### ステップ1: 拡張機能を再読み込み

```
1. chrome://extensions/ を開く
2. 「TheOption Trend Analyzer」の「再読み込み」ボタンをクリック
3. TheOptionのページをリロード（Ctrl+R または Cmd+R）
```

### ステップ2: 詳細ログを確認

TheOptionのページを開いて、コンソールで以下のログを探す:

```
[ML] 🔬 ========== 上位5件のスコア詳細分析 ==========
[ML] 🔬 [1] 類似度: XX% (XX.X/100点)
[ML] 🔬     テクニカル指標: XX.X/60点 | RSI=X.X MACD=X.X ROC=X.X MA=X.X Stoch=X.X ADX=X.X
[ML] 🔬     価格パターン: XX.X/40点 | トレンド=X/15(NEUTRAL⇄NEUTRAL) 傾き=X/10(ΔX.XXX) ...
```

**このログがない場合** → 拡張機能が再読み込みされていない

### ステップ3: スコア内訳を分析

上記ログから以下を確認:

#### チェックポイント1: NEUTRAL同士のスコア
```
トレンド=15/15(NEUTRAL⇄NEUTRAL)  ❌ 修正が反映されていない（15点は異常）
トレンド=3/15(NEUTRAL⇄NEUTRAL)   ✅ 修正が反映されている（3点が正常）
```

#### チェックポイント2: テクニカル指標スコア
```
テクニカル指標: 55.0/60点  ❌ 高すぎる（NEUTRAL同士なら15点前後が正常）
テクニカル指標: 15.0/60点  ✅ 正常範囲
```

NEUTRAL同士のデータで55点以上取っている場合、velocity/volatility/rangeで高得点を獲得している。

#### チェックポイント3: 価格パターンの詳細
```
傾き=10/10(Δ0.001)        → ほぼ同じ傾き（満点）
上昇率=7/7(Δ0.005)        → ほぼ同じ上昇率（満点）
Vol=5/5(Δ0.008)           → ほぼ同じボラティリティ（満点）
```

低ボラティリティ相場では、これらが全て満点近くになるのは正常。

### ステップ4: 学習データ品質を分析

TheOptionのコンソールで以下のスクリプトを実行:

```javascript
(async function analyzeData() {
  console.log('🔍 学習データの分析を開始します...');

  const result = await chrome.storage.local.get(['theoption_ml_BTC_JPY']);
  const trainingData = result.theoption_ml_BTC_JPY || [];

  console.log(`📊 総データ数: ${trainingData.length}件\n`);

  if (trainingData.length === 0) {
    console.log('❌ 学習データが見つかりませんでした');
    return;
  }

  // トレンド方向の分布（15秒）
  const trendDistribution = { UP: 0, DOWN: 0, NEUTRAL: 0, undefined: 0 };

  trainingData.forEach(data => {
    const pattern = data.pricePattern15s;
    if (!pattern) {
      trendDistribution.undefined++;
    } else {
      const direction = pattern.trendDirection || 'undefined';
      trendDistribution[direction] = (trendDistribution[direction] || 0) + 1;
    }
  });

  console.log('📈 トレンド方向の分布 (15秒):');
  console.log(`  UP:       ${trendDistribution.UP} 件 (${(trendDistribution.UP/trainingData.length*100).toFixed(1)}%)`);
  console.log(`  DOWN:     ${trendDistribution.DOWN} 件 (${(trendDistribution.DOWN/trainingData.length*100).toFixed(1)}%)`);
  console.log(`  NEUTRAL:  ${trendDistribution.NEUTRAL} 件 (${(trendDistribution.NEUTRAL/trainingData.length*100).toFixed(1)}%)`);
  console.log(`  未定義:   ${trendDistribution.undefined} 件\n`);

  // ボラティリティ統計
  const volatilityValues = [];
  const trendSlopeValues = [];

  trainingData.forEach(data => {
    const pattern = data.pricePattern15s;
    if (pattern) {
      if (pattern.volatility !== undefined) {
        volatilityValues.push(pattern.volatility);
      }
      if (pattern.trendSlope !== undefined) {
        trendSlopeValues.push(Math.abs(pattern.trendSlope));
      }
    }
  });

  const avgVolatility = volatilityValues.reduce((a, b) => a + b, 0) / volatilityValues.length;
  const avgSlope = trendSlopeValues.reduce((a, b) => a + b, 0) / trendSlopeValues.length;

  console.log('📉 価格変動の統計 (15秒):');
  console.log(`  平均ボラティリティ: ${avgVolatility.toFixed(4)}`);
  console.log(`  平均トレンド傾き: ${avgSlope.toFixed(4)}\n`);

  // 診断
  console.log('💡 診断結果:');

  const neutralRatio = trendDistribution.NEUTRAL / trainingData.length;
  if (neutralRatio > 0.7) {
    console.log(`  ⚠️  NEUTRAL比率が異常に高い (${(neutralRatio*100).toFixed(1)}%)`);
    console.log('     → データ収集タイミングに偏りがある可能性');
    console.log('     → NEUTRAL判定の閾値が甘すぎる可能性');
  }

  if (avgVolatility < 0.05) {
    console.log(`  ⚠️  平均ボラティリティが低い (${avgVolatility.toFixed(4)})`);
    console.log('     → 動きの少ない相場データばかりが保存されている');
    console.log('     → 全てのパターンが類似して見える原因');
  }

  if (avgSlope < 0.05) {
    console.log(`  ⚠️  平均トレンド傾きが低い (${avgSlope.toFixed(4)})`);
    console.log('     → 明確なトレンドのあるデータが少ない');
    console.log('     → 全てのパターンが類似して見える原因');
  }

  console.log('\n✅ 分析完了');
})();
```

## よくある質問

### Q1: 修正後も75%のパターンが類似と判定されるのは正常？

**A**: 状況によります。

- **NEUTRAL比率が70%以上**: 異常。データ収集に偏りがある
- **NEUTRAL比率が50%以下 + 低ボラティリティ**: 正常。実際に全てのパターンが似ている
- **詳細ログでトレンド=3/15**: 修正は反映されている
- **詳細ログでトレンド=15/15**: 修正が反映されていない

### Q2: 低ボラティリティ相場で高一致率なら、どうすればいい？

**A**: 以下の対処法があります。

1. **閾値を上げる**: 90%以上に設定して、本当に似ているパターンのみを抽出
2. **動きのある時間帯を狙う**: 東京時間、ロンドン時間、NY時間の重複時間帯
3. **データ収集を継続**: 動きのある相場データが蓄積されるまで待つ

### Q3: NEUTRAL同士で3点しか取れないのに、なぜ75%も一致する？

**A**: 価格パターンの他の要素が満点を取っているためです。

```
トレンド一致: 3/15    ← NEUTRAL減点
傾き: 10/10           ← 満点（低ボラティリティで同じ傾き）
上昇率: 7/7           ← 満点（低ボラティリティで同じ上昇率）
ボラティリティ: 5/5   ← 満点（全て低ボラティリティ）
パターンタイプ: 3/3   ← 満点（全てRANGEまたはNEUTRAL）

合計: 28/40点 (70%)
```

テクニカル指標も velocity≈0, volatility≈小, range≈小 で類似すると、40/60点程度取れます。

合計: 68/100点 = **68%類似度**

これは低ボラティリティ相場では正常な動作です。
