# 修正完了：即座のデータ保存システム（Pending方式）

## 🔴 発生していた問題

**症状**: 10件収集してブラウザ更新すると、データが0件からスタートする

**根本原因**:
- データ収集時に`result15s`等はnull
- setTimeout（15秒後、30秒後...）で結果を記録する仕組み
- ブラウザ更新でsetTimeoutが消失
- 価格履歴からの復元も、履歴が短いと不可能
- 結果: 10件収集→更新→使えるデータ0件

**ユーザーの期待**:
- 10件収集したら、更新後も10件のデータが使える
- データ収集が無駄にならない

---

## ✅ 実装した解決策

### アイデア: Pending（保留中）方式

**従来の方式**:
```
データ収集時:
  result15s: null  ← 結果なし

15秒後（setTimeout）:
  result15s: { price: 100.25, ... }  ← 結果記録

ブラウザ更新:
  setTimeoutが消失 → 結果が記録されない
```

**新しい方式**:
```
データ収集時:
  result15s: {
    price: 100.245,  ← 現在価格
    change: 0,
    direction: 'NEUTRAL',
    pending: true    ← 未確定フラグ
  }
  即座に保存！

15秒後（setTimeout）:
  result15s: {
    price: 100.250,  ← 実際の価格
    change: +0.005,
    direction: 'UP',
    pending: false   ← 確定済み
  }
  上書き保存！

ブラウザ更新:
  pendingデータがあるが、使える！
  （予測では pending=true はスキップ）
```

### 1. recordSituation()を修正（即座保存）

**machine-learning-system.js Line 20-104:**

```javascript

```

**変更点**:

### 2. recordResult()を修正（上書き更新）

**machine-learning-system.js Line 117-142:**

```javascript
recordResult(situation, seconds) {
  const currentPrice = window.theOptionCurrentPrice || situation.price;
  const change = currentPrice - situation.price;
  const changePercent = (change / situation.price) * 100;
  const direction = change > 0 ? 'UP' : change < 0 ? 'DOWN' : 'NEUTRAL';

  // 実際の結果で上書き（pendingフラグを削除）
  situation[`result${seconds}s`] = {
    price: currentPrice,
    change: change,
    changePercent: changePercent,
    direction: direction,
    pending: false  // ← 確定済み
  };

  // 全タイムフレームの結果が確定したら保存
  const allConfirmed = [15, 30, 60, 180, 300].every(tf =>
    situation[`result${tf}s`] && !situation[`result${tf}s`].pending
  );

  if (allConfirmed) {
    console.log(`[ML] データID ${situation.timestamp}: 全結果確定 - 保存`);
    this.saveToStorage();
  }
}
```

**変更点**:
- プレースホルダーを**実際の値で上書き**
- `pending: false`で確定済みとマーク
- 全結果が確定したら保存

findSimilarPatterns(currentSituation, timeframe = 15, minSimilarity = 70) {
  const similarPatterns = [];

  for (const past of this.trainingData) {
    const result = past[`result${timeframe}s`];
    if (!result) continue;

    // pending（未確定）のデータはスキップ
    if (result.pending === true) continue;  // ← 追加

        pattern: past,
        similarity: similarity,
        result: result
      });
    }
  }

  return similarPatterns;
}

## 📝 技術的な詳細

### Pendingフラグの役割

**pending: true**
- 意味: データは保存されているが、結果は未確定
- 用途: ストレージには保存される、予測には使用されない
- いつセット: データ収集時

**pending: false（または undefined）**
- 意味: 結果が確定済み
- 用途: 予測に使用可能
- いつセット: setTimeout完了後、または価格履歴から復元後

---

### データの状態遷移

```
[収集時]
  result15s: { price: 100.245, pending: true }
  ↓ 即座に保存

[15秒後]
  result15s: { price: 100.250, pending: false }
  ↓ 上書き保存

[ブラウザ更新]
  ↓ データ読み込み

[pending=true の場合]
  ↓ 価格履歴から復元を試みる

[復元成功]
  result15s: { price: 100.248, pending: false }
  ↓ 保存

[復元失敗]
  result15s: { price: 100.245, pending: true }
  ↓ そのまま（予測には使用されない）

[新しいデータ収集]
  15秒経過 → pending=false に更新
  ↓ 使用可能になる
```

---

### 保存頻度の最適化

**従来**: 100件ごと、または全結果確定時
**新方式**: 10件ごと + 全結果確定時

**理由**:
- 10件ごとに保存することで、こまめにデータを永続化
- ブラウザクラッシュやタブ終了でもデータ損失を最小化
- ストレージ負荷は増えるが、データ保護が優先
