# 修正完了：モード切替時のデータ保持機能

## 🔴 発生していた問題

**症状**: リアルタイムモード ⇄ デモモード間で切り替えると、全てのデータが初期化される

**具体的な状況**:
1. リアルタイムモードで AUD/JPY のデータを120秒以上収集
2. デモモードに移動
3. 再びリアルタイムモードに戻る
4. 「データ収集を開始」と表示され、AIの学習データも0からスタート

**ユーザーの期待**:
- 保存されているはずのデータが消えずに復元される
- モード切替前のAI学習データが引き継がれる

---

## 📋 問題の根本原因

### 1. 通貨ペア名の不一致

TheOptionのサイトでは、モードによって通貨ペアの表示名が異なる可能性があります：

| モード | 表示名の例 |
|--------|-----------|
| リアルタイム | `AUD/JPY` |
| デモ | `AUD/JPY (Demo)` または `AUD/JPY デモ` |

### 2. データ保存のキー生成

拡張機能は通貨ペア名をキーにしてデータを保存します：

```javascript
const storageKey = `theoption_price_${asset.replace(/[\/\s]/g, '_')}`;

// リアルタイム: theoption_price_AUD_JPY
// デモ:         theoption_price_AUD_JPY__Demo_  ← 異なるキー！
```

### 3. データ復元の失敗

`theoption-analyzer.js` Line 997:
```javascript
const savedData = await loadPriceData(detectedAsset);
if (savedData) {
  // データを復元
} else {
  // データが見つからない → 初期化！
  priceHistory = [];
  tickData = [];
  candles = [];
}
```

**問題**: `AUD/JPY (Demo)` で保存したデータは、`AUD/JPY` では見つからない → 初期化される

---

## ✅ 実装した解決策

### 通貨ペア名正規化システム

**アイデア**: どのモードでも同じ通貨ペア名として扱う

```
AUD/JPY (Demo)  ──┐
AUD/JPY デモ    ──┼──→ 正規化 → AUD/JPY
AUD/JPY [リアル] ──┘
```

## 🔧 実装内容

### 1. 正規化関数の追加（Line 1962-1977）

```javascript
/**
 * 通貨ペア名を正規化（モード表示を除去）
 * 例: "AUD/JPY (Demo)" → "AUD/JPY"
 *     "AUD/JPY デモ" → "AUD/JPY"
 *     "AUD/JPY [リアル]" → "AUD/JPY"
 */
function normalizeAssetName(assetName) {
  if (!assetName) return assetName;

  // モード表示を除去するパターン
  return assetName
    .replace(/\s*[\(（].*?[\)）]\s*/g, '')  // (Demo), （デモ）など
    .replace(/\s*[\[【].*?[\]】]\s*/g, '')  // [リアル], 【デモ】など
    .replace(/\s*(demo|デモ|real|リアル|practice|プラクティス)\s*$/gi, '')  // 末尾のモード表示
    .trim();
}
```

**対応パターン**:
- `AUD/JPY (Demo)` → `AUD/JPY`
- `AUD/JPY （デモ）` → `AUD/JPY`
- `AUD/JPY [リアル]` → `AUD/JPY`
- `AUD/JPY 【デモ】` → `AUD/JPY`
- `AUD/JPY demo` → `AUD/JPY`
- `AUD/JPY デモ` → `AUD/JPY`
- `AUD/JPY practice` → `AUD/JPY`

## 📝 技術的な詳細

### なぜ全ての検出メソッドに適用したのか？

TheOptionサイトでは、以下の複数の方法で通貨ペアを検出します：

1. **activeクラス要素**: ユーザーが選択中の通貨ペア
2. **data属性**: `data-asset`, `data-pair` など
3. **URLパラメータ**: `?asset=AUDJPY` など
4. **検出リスト**: セレクタで見つかった全候補から選択
5. **価格要素の親**: 価格表示の近くにある通貨ペア名

**どの方法でもモード表示が含まれる可能性があるため、全てに正規化を適用しました。**

---

### ストレージキーの仕組み

```javascript
// 通貨ペア名: "AUD/JPY (Demo)"
const asset = normalizeAssetName("AUD/JPY (Demo)");  // → "AUD/JPY"
const storageKey = `theoption_price_${asset.replace(/[\/\s]/g, '_')}`;
// → "theoption_price_AUD_JPY"

// 通貨ペア名: "AUD/JPY"
const asset = normalizeAssetName("AUD/JPY");  // → "AUD/JPY"
const storageKey = `theoption_price_${asset.replace(/[\/\s]/g, '_')}`;
// → "theoption_price_AUD_JPY"

✅ 同じキーになる！
```

### 機械学習データも保持される

`machine-learning-system.js` では通貨ペア名をキーに使用：

```javascript
mlSystem.setCurrentAsset(detectedAsset);  // 正規化後の名前が渡される
mlSystem.initialize(detectedAsset);  // → "theoption_ml_AUD_JPY"
```

**結果**: MLデータも正しく復元される
