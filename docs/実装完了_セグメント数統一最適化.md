# セグメント数統一最適化 実装完了

## 実装日時
2025-11-02

## 問題点

**セグメント数の増加による一致率の低下**:

### 変更前の設定

| 判定時間 | セグメント数 | 理論的パターン数 | 一致の難易度 |
|---------|------------|----------------|------------|
| 15秒 | 6個 | 3^6 = **729通り** | 易 ✅ |
| 30秒 | 9個 | 3^9 = **19,683通り** | 中 ⚠️ |
| 60秒 | 12個 | 3^12 = **531,441通り** | 難 ❌ |
| 180秒 | 12個 | 3^12 = **531,441通り** | 難 ❌ |
| 300秒 | 15個 | 3^15 = **14,348,907通り** | 超難 ❌❌ |

### 問題の詳細

1. **一致確率の指数関数的減少**
   - セグメント数が増えるほど、完全一致するパターンを見つけるのが困難
   - 3分・5分判定では、学習データ10,000件でもほとんど一致しない

2. **予測精度への影響**
   - 長期判定で「データ不足」が頻発
   - 類似パターンが見つからず、予測不能になる

3. **学習効率の低下**
   - せっかく蓄積したデータが活用されない
   - 長期判定の予測精度が上がらない

## 解決策

### **全判定時間を6セグメントに統一**

**コンセプト**: パターン数を統一し、全判定時間で同程度の一致率を確保

## 理論的根拠

### なぜ6セグメントが最適か？

**1. パターン認識の観点**:
- 人間の視覚的パターン認識能力: 5-9個の要素を一度に把握可能
- 6個は「ちょうど良い粒度」で、形状を十分に表現できる

**2. 統計的な観点**:
- 3^6 = 729通り
- 学習データ100件で各パターン約0.14回
- 学習データ1,000件で各パターン約1.4回
- 学習データ10,000件で各パターン約13.7回 ✅

**3. 計算効率の観点**:
- セグメント数が少ない → 計算が高速
- リアルタイム分析に適している

**4. 機械学習の観点**:
- 特徴量が多すぎると過学習のリスク
- 6個の特徴量は適度な抽象化レベル
