# TheOption Trend Analyzer 改善設計書

## 🎯 現状の問題点

### 1. シグナルが1秒ごとにコロコロ変わる
**原因**:
- 1秒ごとに新しい価格データを追加
- 毎秒分析を実行して即座にUI更新
- 短期的なノイズ（価格の微小な変動）に過敏に反応

**影響**:
- ユーザーが判断できない
- 信頼性が低く感じる
- 実用性がない

### 2. 過去データが取得できない
**調査結果**:
- ❌ REST APIで過去ローソク足データを取得するエンドポイントは**見つかりませんでした**
- ⚠️ Lightstreamerはリアルタイムストリーミング専用
- ⚠️ チャートのcanvas要素から直接データ抽出も困難

**現状の制約**:
- ページ表示後にリアルタイムでデータを蓄積するしかない
- 50ポイント（50秒）待たないと分析開始できない

---

## 💡 解決策の設計

### アプローチ: 「安定したトレンド分析システム」

過去データが取得できない制約の中で、**信頼性の高いシグナル**を提供する。

## 🔧 具体的な改善策

### 改善1: マルチタイムフレーム分析

**概念**:
異なる時間足で分析し、複数の視点からトレンドを確認

```
┌─────────────────────────────────────┐
│ 短期 (1分足)  → 直近のモメンタム  │
│ 中期 (5分足)  → トレンド方向      │
│ 長期 (15分足) → 大局的な流れ      │
└─────────────────────────────────────┘
```

**実装**:
```javascript
class MultiTimeframeAnalyzer {
  constructor() {
    this.tf1m = new CandleBuilder(60);   // 1分足
    this.tf5m = new CandleBuilder(300);  // 5分足
    this.tf15m = new CandleBuilder(900); // 15分足
  }

  addTick(price, timestamp) {
    // 全てのタイムフレームに価格を追加
    this.tf1m.addTick(price, timestamp);
    this.tf5m.addTick(price, timestamp);
    this.tf15m.addTick(price, timestamp);
  }

  analyze() {
    const analysis1m = this.analyzeTF(this.tf1m.getCandles(), '1分');
    const analysis5m = this.analyzeTF(this.tf5m.getCandles(), '5分');
    const analysis15m = this.analyzeTF(this.tf15m.getCandles(), '15分');

    // 複数時間足で一致した場合のみ強いシグナル
    return this.combineSignals([analysis1m, analysis5m, analysis15m]);
  }
}
```

**メリット**:
- 短期的なノイズに惑わされない
- 大局的なトレンドと短期的な動きの両方を見る
- 複数時間足で一致 = 信頼度UP

### 改善2: シグナル安定化メカニズム

#### 2-1. 連続確認方式

**概念**:
同じシグナルが連続N回確認されて初めて表示

```javascript
class SignalStabilizer {
  constructor(confirmCount = 3) {
    this.confirmCount = confirmCount;
    this.history = [];
  }

  addSignal(signal) {
    this.history.push(signal);
    if (this.history.length > this.confirmCount) {
      this.history.shift();
    }

    // 連続して同じシグナルか確認
    if (this.history.length === this.confirmCount) {
      const allSame = this.history.every(s => s === this.history[0]);
      if (allSame) {
        return { confirmed: true, signal: this.history[0] };
      }
    }

    return { confirmed: false, signal: null };
  }
}
```

**例**:
```
時刻  | 分析結果 | 表示    | 説明
------|---------|---------|------------------
0秒   | HIGH    | WAIT    | 1回目、まだ確定せず
1秒   | HIGH    | WAIT    | 2回目、まだ確定せず
2秒   | HIGH    | ✅ HIGH | 3回連続 → 確定!
3秒   | LOW     | HIGH    | 変化したが、連続3回まで保持
4秒   | LOW     | HIGH    | 2回目
5秒   | LOW     | ✅ LOW  | 3回連続 → 変更確定
```

#### 2-2. 閾値ベース判定

**概念**:
スコアが一定の閾値を超えないとシグナルを出さない

```javascript
determineTrend(score) {
  // 強い閾値設定
  if (score >= 5) {
    return { signal: 'HIGH', confidence: 80 };
  } else if (score <= -5) {
    return { signal: 'LOW', confidence: 80 };
  } else {
    // 中途半端なスコアは「様子見」
    return { signal: 'NEUTRAL', confidence: 40 };
  }
}
```

**現在との違い**:
- 現在: スコア±2でHIGH/LOW判定 → 頻繁に変わる
- 改善後: スコア±5でHIGH/LOW判定 → 明確なトレンドのみ

### 改善3: トレンド継続性の評価

**概念**:
トレンドの「強さ」と「持続性」を評価

```javascript
class TrendAnalyzer {
  constructor() {
    this.currentTrend = null;
    this.trendStartTime = null;
    this.trendDuration = 0;
  }

  updateTrend(signal) {
    if (signal !== this.currentTrend) {
      // トレンド変化
      this.currentTrend = signal;
      this.trendStartTime = Date.now();
      this.trendDuration = 0;
    } else {
      // トレンド継続
      this.trendDuration = Date.now() - this.trendStartTime;
    }

    // トレンドが30秒以上継続 → 信頼度UP
    const bonus = this.trendDuration > 30000 ? 10 : 0;

    return {
      signal: this.currentTrend,
      duration: this.trendDuration,
      confidenceBonus: bonus
    };
  }
}
```

**メリット**:
- 長く続いているトレンド = 信頼性高い
- 短命なシグナル = 無視

### 改善4: ボラティリティフィルター

**概念**:
価格変動が小さい時はシグナルを出さない

```javascript
calculateVolatility(prices, period = 20) {
  const recentPrices = prices.slice(-period);
  const avg = recentPrices.reduce((a, b) => a + b) / period;
  const variance = recentPrices.reduce((sum, p) => sum + Math.pow(p - avg, 2), 0) / period;
  const stdDev = Math.sqrt(variance);

  return stdDev;
}

shouldTrade(volatility, minVolatility = 10) {
  if (volatility < minVolatility) {
    return {
      trade: false,
      reason: 'ボラティリティ不足（レンジ相場）'
    };
  }
  return { trade: true };
}
```

**効果**:
- レンジ相場（横ばい）= シグナル出さない
- トレンド相場（大きな動き）= シグナル出す

### 改善5: 疑似ローソク足生成による過去データ活用

**概念**:
1秒ごとのTick価格から、1分足・5分足のローソク足を生成

```javascript
class CandleBuilder {
  constructor(timeframe = 60) {
    this.timeframe = timeframe; // 秒
    this.currentCandle = null;
    this.completedCandles = [];
  }

  addTick(price, timestamp) {
    const candleTime = Math.floor(timestamp / this.timeframe) * this.timeframe;

    if (!this.currentCandle || this.currentCandle.time !== candleTime) {
      // 前のローソク足を完成させる
      if (this.currentCandle) {
        this.completedCandles.push(this.currentCandle);
        if (this.completedCandles.length > 100) {
          this.completedCandles.shift(); // 古いデータ削除
        }
      }

      // 新しいローソク足を開始
      this.currentCandle = {
        time: candleTime,
        open: price,
        high: price,
        low: price,
        close: price,
        ticks: 1
      };
    } else {
      // 現在のローソク足を更新
      this.currentCandle.high = Math.max(this.currentCandle.high, price);
      this.currentCandle.low = Math.min(this.currentCandle.low, price);
      this.currentCandle.close = price;
      this.currentCandle.ticks++;
    }
  }

  getCandles() {
    // 完成したローソク足 + 現在進行中のローソク足
    return this.currentCandle
      ? [...this.completedCandles, this.currentCandle]
      : this.completedCandles;
  }
}
```

**メリット**:
- 1分で分析開始可能（1分足の1本目が完成した瞬間）
- 5分待てば5分足で分析可能
- より正確なトレンド判定

## 📊 新しい分析ロジック

### 総合判定フロー

```
1. データ収集（最低1分）
   ↓
2. ローソク足生成（1分足/5分足/15分足）
   ↓
3. 各タイムフレームで分析
   ├─ MA分析
   ├─ RSI分析
   ├─ ボリンジャーバンド分析
   └─ MACD分析
   ↓
4. スコア計算（各タイムフレーム）
   ↓
5. マルチタイムフレーム統合
   ├─ 3つの時間足で一致？
   ├─ スコアが閾値超え？
   └─ ボラティリティ十分？
   ↓
6. トレンド継続性チェック
   ├─ 同じシグナルが3回連続？
   └─ トレンド持続時間は？
   ↓
7. 最終シグナル出力
   ├─ HIGH (信頼度70-90%)
   ├─ LOW (信頼度70-90%)
   └─ NEUTRAL/WAIT (トレンド不明瞭)
```

## ⚠️ 制約事項

### 避けられない制約
1. **初回起動時は待機必要**: 最低3分（15分足1本完成）
2. **過去データ無し**: API提供されていないため取得不可
3. **リアルタイム依存**: ページ表示中のみデータ蓄積

### 代替案
- **localStorage活用**: 一度収集したデータを保存 → 次回即時分析
- **バックグラウンド実行**: Service Worker で継続的にデータ蓄積
