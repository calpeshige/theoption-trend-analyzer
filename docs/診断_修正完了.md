# 類似度計算バグ修正 - 完了報告

## 発見された問題

### 1. テクニカル指標の2重加算バグ (重大)

**問題**: [machine-learning-system.js:652-726](machine-learning-system.js#L652-L726)で、セグメント分析がない場合のフォールバック処理でテクニカル指標のスコアを加算した後、さらに同じテクニカル指標を再度加算していました。

**影響**:
- セグメント分析あり → 正常動作 (価格40点 + テクニカル30点 + 従来型30点 = 100点)
- セグメント分析なし → **異常動作** (価格40点 + テクニカル30点×2 = 100点)

**実際の影響**:
- 全てのデータでセグメント分析が欠落している場合、テクニカル指標のスコアが60点満点で計算される
- これにより、テクニカル指標が類似しているデータが過剰に高スコアを獲得
- ユーザーの報告通り、ほとんどのデータが類似と判定される原因

### 2. デバッグ情報の形式不一致

**問題**: 旧方式のデバッグ情報が`legacyTrend`, `legacyPattern`という名前で保存されていたが、デバッグ出力では`trendMatch`, `slope`などを期待していました。

**影響**:
- デバッグ出力でエラーが発生(TypeError)
- 類似度の内訳が正しく表示されない

## 修正内容

### 修正1: テクニカル指標の2重加算を削除

**変更前**:
```javascript
// セグメント分析がない場合
if (currentTechSegments && pastTechSegments) {
  // セグメント分析を使用 (30点)
} else {
  // フォールバック: 従来型 (30点)
  score += rsiScore + macdScore + ...;
}

// === さらに従来型を加算 (30点) === ← 2重加算!
score += rsiScore + macdScore + ...;
```

**変更後**:
```javascript
// === テクニカル指標比較 (30点) ===
// 注: techSegmentsは将来の拡張用。現在は従来型比較のみ実装
const rsiScore = this.compareTechIndicator(...);
const macdScore = this.compareTechIndicator(...);
score += rsiScore + macdScore + ...;  // ← 1回のみ
```

### 修正2: デバッグ情報の形式を統一

**変更前**:
```javascript
debugInfo.details.legacyTrend = { points: trendPoints, max: 15 };
debugInfo.details.legacyPattern = { slope: slopePoints, ... };
```

**変更後**:
```javascript
debugInfo.details.trendMatch = {
  points: trendPoints,
  current: currentPattern.trendDirection,
  past: pastPattern.trendDirection
};
debugInfo.details.slope = {
  points: slopePoints,
  diff: slopeDiff.toFixed(3)
};
// ... 他の指標も同様
```

### 修正3: priceSegmentScoreの追加

旧方式でも価格パターンスコアを`debugInfo.priceSegmentScore`に記録するように変更:

```javascript
const legacyPriceScore = trendPoints + slopePoints + upRatioPoints + volPoints + patternTypePoints;
debugInfo.priceSegmentScore = legacyPriceScore;
```

### ステップ3: セグメント分析の確認

コンソールで以下のスクリプトを実行:

```javascript
(async function() {
  const result = await chrome.storage.local.get(['theoption_ml_BTC_JPY']);
  const data = result.theoption_ml_BTC_JPY || [];

  let withSeg = 0;
  let withoutSeg = 0;

  data.forEach(d => {
    if (d.priceSegments15s) withSeg++;
    else withoutSeg++;
  });

  console.log(`総データ: ${data.length}件`);
  console.log(`セグメント分析あり: ${withSeg}件`);
  console.log(`セグメント分析なし: ${withoutSeg}件`);

  if (data.length > 0) {
    const latest = data[data.length - 1];
    console.log('最新データにpriceSegments15s:', latest.priceSegments15s ? 'あり' : 'なし');
    if (latest.priceSegments15s) {
      console.log('セグメント数:', latest.priceSegments15s.segmentCount);
      console.log('パターン:', latest.priceSegments15s.pattern);
      console.log('要約:', latest.priceSegments15s.summary);
    }
  }
})();
```

**期待される結果**:
- 最新データには`priceSegments15s`が含まれているはず
- セグメント数は6 (15秒判定の場合)
- パターンは`UP-DOWN-FLAT`のような形式

## トラブルシューティング

### 問題1: まだ全て同じスコアが表示される

**原因**: 古いデータのみが保存されている
**解決策**:
1. 新しいデータが収集されるまで待つ (15秒ごとに1件)
2. または、ストレージをクリアして新データを収集:
   ```javascript
   chrome.storage.local.remove(['theoption_ml_BTC_JPY']);
   ```

### 問題2: セグメント分析データがない

**原因**: `DetailedSegmentAnalyzer`が初期化されていない
**解決策**:
1. ページリロード
2. コンソールで確認:
   ```javascript
   console.log(window.DetailedSegmentAnalyzer);  // クラスが存在するか
   ```

### 問題3: デバッグ出力でエラーが表示される

**原因**: 修正が反映されていない
**解決策**:
1. 拡張機能を完全に削除
2. 再インストール
3. TheOptionページをリロード
