# テクニカル分析エンジン改修要件定義書 (v5.0: TheOption完全対応版)

**プロジェクト名**: High-Frequency Binary Options Analysis System Upgrade
**対象プラットフォーム**: TheOption (15秒, 30秒, 1分, 3分, 5分)
**バージョン**: 5.0.0
**作成日**: 2025年11月27日

---

## 1. 改修の目的と概要

### 1.1 現状の課題
* **抵抗帯での敗北**: 急騰後のV字反転（行って来い）の場面で、既存ロジックが「強いトレンド」と誤認し、天井でHIGHエントリーしてしまう。
* **レンジ相場の機会損失**: ADXによる足切りが厳しすぎ、ボックストレンド（レンジ）での逆張り機会を全て捨てている。
* **短期判定の遅延**: 15秒・30秒取引において、MACDやADXなどの遅行指標が機能せず、判定負けが発生している。

### 1.2 改修のゴール
1. **「環境認識」の導入**: 相場を「トレンド」と「レンジ」に分類し、ロジックを動的に切り替える。
2. **「抵抗帯フィルター」の実装**: 直近高値・安値付近での順張りエントリーを物理的にブロックする。
3. **時間枠別最適化**: 15秒～5分の各判定時間に応じたパラメータと使用指標の自動切り替えを実装する。

## 2. システムアーキテクチャ（処理フロー刷新）

従来の「全指標合算スコア（Total Score）」方式を廃止し、以下の条件分岐フローへ移行する。

### 新フローチャート
1. **初期化**: 判定時間(`timeframe`)に応じたConfigのロード
2. **Phase Detection (環境認識)**: 現在が `TREND` か `RANGE` かを判定
3. **Logic Selection (ロジック選択)**:
   * `TREND` → **順張りスコア計算** (MACD, ROC主体 / RSI高値は肯定)
   * `RANGE` → **逆張りスコア計算** (BB, Stoch主体 / RSI高値は否定)
4. **Resistance Filter (安全装置)**: シグナル方向にある「直近の壁」を確認
5. **最終判定**: フィルター通過後、エントリーシグナルを出力

## 3. 指標の役割分担（適材適所）

全指標を常に使うのではなく、フェーズと時間枠によって役割を限定する。

| 指標名 | 役割 | トレンド時 | レンジ時 | 15秒/30秒 | 備考 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Bollinger Bands** | **指揮官 / 兵隊** | トレンド判定 | 逆張り基準(±2σ) | **主力** | 環境認識と逆張りの両方で使用 |
| **ADX** | **指揮官** | トレンド判定 | (使用しない) | **無効** | トレンドスイッチ役。短期では遅すぎるため無視 |
| **RSI** | **副官** | 勢い確認(>50) | 反転確認(>80) | **主力** | フェーズにより「順張り」か「逆張り」か意味が逆転する |
| **MACD** | 兵隊 (順張り) | 主力信号 | **無視** | 無効 | レンジ・超短期ではダマシになるため計算しない |
| **Stochastic** | 兵隊 (逆張り) | **無視** | 主力信号 | 無効 | トレンド中の張り付きを防ぐため、レンジ限定で使用 |
| **ROC** | 兵隊 (瞬発力) | 加点要素 | (使用しない) | **主力** | 短期の「初動」を捉えるために使用 |
| **ATR** | 裏方 | 閾値調整 | 閾値調整 | 閾値調整 | ボラティリティによる動的閾値計算に使用 |
| **Sentiment** | (廃止) | - | - | - | 今回の改修で削除 |

## 4. 時間枠別パラメータ設定 (Dynamic Config)

判定時間(`targetTimeframe`)に応じて、以下の定数を動的に切り替えること。
**※特に15秒/30秒は反応速度を最優先とし、ADX等は使用しない。**

| 設定項目 (変数) | 15秒 (Ultra) | 30秒 (Turbo) | 1分 (Short) | 3分/5分 (Normal) | 解説 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **抵抗帯検索期間**<br>`RESISTANCE_LOOKBACK` | **30本** | **50本** | **60本** | **120本** | 短期ほど直近の壁だけを意識する |
| **抵抗帯接近許容値**<br>`PROXIMITY_THRESHOLD` | **0.01%** | **0.02%** | **0.03%** | **0.05%** | 短期は壁ギリギリまで攻める設定 |
| **トレンド判定(ADX)**<br>`ADX_TREND_MIN` | **無効 (0)** | **無効 (0)** | **30** | **25** | 30秒以下ではADX計算をスキップ |
| **バンド拡大率**<br>`BB_EXPANSION_RATIO` | **1.3倍** | **1.25倍** | **1.2倍** | **1.1倍** | 短期はノイズ除去のため急拡大のみ検知 |
| **レンジ逆張り基準(RSI)**<br>`RSI_OVERBOUGHT` | **90 / 10** | **85 / 15** | **80 / 20** | **75 / 25** | 超短期のRSI70は通過点。異常値のみ狙う |
| **レンジ逆張り基準(Stoch)**<br>`STOCH_OVERBOUGHT` | **無効** | **無効** | **90 / 10** | **80 / 20** | 短期でストキャスは機能しない |

## 5. ロジック実装詳細仕様

### 5.1 環境認識 (Phase Detection)

```javascript
// 入力: config, adx, bb_width, prev_bb_width
let phase = 'RANGE'; // デフォルト

// バンド拡大率の計算
const expansionRate = bb_width / prev_bb_width;

if (timeframe < '1m') {
    // 【15秒・30秒】バンドの急拡大のみでトレンド判定（ADX無視）
    if (expansionRate > config.BB_EXPANSION_RATIO) {
        phase = 'TREND';
    }
} else {
    // 【1分以上】ADXとバンド拡大のAND条件
    if (adx > config.ADX_TREND_MIN && expansionRate > config.BB_EXPANSION_RATIO) {
        phase = 'TREND';
    }
}

5.2 ロジック分岐 (Logic Split)
A. TREND Phase (順張りモード)
概要: 抵抗帯にぶつかるまでトレンドに乗る。

採用指標:

MACD: ゴールデンクロス/デッドクロスで加点 (1分以上のみ)。

ROC: 直近の変化率が大きければ加点。

RSI: 50以上なら加点。70以上でも減点しない（強いトレンドとみなす）。

判定: trendScore > threshold で HIGH 候補。

B. RANGE Phase (逆張りモード)
概要: 加熱しすぎた相場の反転を狙う。

採用指標:

Bollinger Bands: 価格が±2σタッチで加点、±3σ到達で特大加点。

RSI: config.RSI_OVERBOUGHT を超えたら LOW方向 へ加点。

Stochastic: 80超えでLOW方向、20未満でHIGH方向へ加点 (1分以上のみ)。

判定: reversalScore > threshold で LOW 候補。

5.3 抵抗帯フィルター (Resistance Filter)
最重要実装項目。シグナル確定直前に実行する。

// 入力: 候補シグナル, 過去価格配列(prices), config
const lookbackPrices = prices.slice(-config.RESISTANCE_LOOKBACK);
const resistance = Math.max(...lookbackPrices); // 直近最高値
const support = Math.min(...lookbackPrices);    // 直近最安値
const currentPrice = prices[prices.length - 1];

// 接近判定閾値 (価格 * %)
const limitDist = currentPrice * config.PROXIMITY_THRESHOLD;

if (signal === 'HIGH') {
    // 上に天井(抵抗)があり、かつ距離が近すぎる場合
    if ((resistance - currentPrice) < limitDist && (resistance > currentPrice)) {
        console.log(`[FILTER] Resistance at ${resistance}. Entry Cancelled.`);
        return 'NEUTRAL';
    }
}
else if (signal === 'LOW') {
    // 下に底(支持)があり、かつ距離が近すぎる場合
    if ((currentPrice - support) < limitDist && (support < currentPrice)) {
        console.log(`[FILTER] Support at ${support}. Entry Cancelled.`);
        return 'NEUTRAL';
    }
}

6. 既存コードの移行・削除指示
競合（コンフリクト）回避のため、以下の修正を徹底すること。

削除 (Delete):

totalScore による単純合算ロジック全体。

if (trendConfidence > 7) などの ADX値を必須とする足切り条件（レンジエントリー阻害の要因）。

Sentiment 指標に関する記述。

維持 (Keep):

各テクニカル指標（MACD, RSI, BB等）の数値計算関数自体。

ATRによるボラティリティ計算ロジック。

データソース:

15秒/30秒判定時は、可能であれば 1分足 ではなく 5秒足 または ティックデータ を使用して計算すること。

不可の場合は1分足を使用するが、RSI等の期間を極端に短く（期間3など）設定すること。
