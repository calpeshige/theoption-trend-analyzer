## 1. 改修の目的と概要

### 1.1 現状の課題
* **抵抗帯での敗北**: 急騰後のV字反転（行って来い）の場面で、既存ロジックが「強いトレンド」と誤認し、天井でHIGHエントリーしてしまう。
* **レンジ相場の機会損失**: ADXによる足切りが厳しすぎ、ボックストレンド（レンジ）での逆張り機会を全て捨てている。
* **短期判定の遅延**: 15秒・30秒取引において、MACDやADXなどの遅行指標が機能せず、判定負けが発生している。

### 1.2 改修のゴール
1. **「環境認識」の導入**: 相場を「トレンド」と「レンジ」に分類し、ロジックを動的に切り替える。
2. **「抵抗帯フィルター」の実装**: 直近高値・安値付近での順張りエントリーを物理的にブロックする。
3. **時間枠別最適化**: 15秒～5分の各判定時間に応じたパラメータと使用指標の自動切り替えを実装する。

## 3. PhaseDetector 詳細仕様

### 3.1 判定ロジック

```javascript
// 超短期 (15秒/30秒): BB拡張率のみで判定
if (timeframeSeconds <= 30) {
  if (bbExpansionRate > trendThreshold.bbExpansion) {
    phase = 'TREND';
  } else if (bbExpansionRate < rangeThreshold.bbExpansion) {
    phase = 'RANGE';
  } else {
    phase = 'TRANSITION';
  }
}

// 長期 (60秒以上): BB拡張率 + ADX
else {
  const isTrendByBB = bbExpansionRate > trendThreshold.bbExpansion;
  const isTrendByADX = adxResult.adx > trendThreshold.adx;

  if (isTrendByBB && isTrendByADX) {
    phase = 'TREND';
  } else if (isRangeByBB && isRangeByADX) {
    phase = 'RANGE';
  }
}
```

if (phase === 'TREND') {

      prices[prices.length - 1] > bbResult.middle) {

## 5. ResistanceFilter 詳細仕様

### 5.1 フィルタリングロジック

```javascript
// 直近50本のローソク足から高値・安値を取得
const highestHigh = Math.max(...recentCandles.map(c => c.high));
const lowestLow = Math.min(...recentCandles.map(c => c.low));
const priceRange = highestHigh - lowestLow;

// 抵抗帯閾値 = 価格レンジの5%
const resistanceThreshold = priceRange * 0.05;

// HIGHシグナル + 天井付近 → ブロック
if ((signal === 'HIGH' || signal === 'STRONG_HIGH') &&
    currentPrice >= (highestHigh - resistanceThreshold)) {
  return { blocked: true, reason: 'NEAR_RESISTANCE' };
}

// LOWシグナル + 底付近 → ブロック
if ((signal === 'LOW' || signal === 'STRONG_LOW') &&
    currentPrice <= (lowestLow + resistanceThreshold)) {
  return { blocked: true, reason: 'NEAR_SUPPORT' };
}
```

### 5.2 ブロック例

```
例: USD/JPY 現在価格 150.50

直近50本の高値: 150.60
直近50本の安値: 150.20
価格レンジ: 0.40 (40pips)
閾値: 0.02 (2pips = 5%)

HIGHシグナル発生時:
  - 天井との距離: 150.60 - 150.50 = 0.10 (10pips)
  - 閾値(2pips) < 距離(10pips) → 通過 ✅

HIGHシグナル発生時 (価格=150.58):
  - 天井との距離: 150.60 - 150.58 = 0.02 (2pips)
  - 閾値(2pips) >= 距離(2pips) → ブロック ❌
```

## 7. 削除された機能

### 7.1 トレンド強度フィルター (UI設定)

**削除理由**: V2アーキテクチャではPhaseDetectorが動的にTREND/RANGEを判定するため、ユーザーが手動で設定する必要がなくなった。

**削除ファイル**:
- `sidepanel.html`: トレンド強度フィルターのドロップダウン
- `sidepanel.js`: 設定の読み込み・保存・イベント処理
- `theoption-analyzer.js`: trendStrengthFilter変数、trendStrengthThresholds定数

### 7.2 Sentiment指標（短期での使用）

**変更内容**: 180秒（3分）未満の時間枠ではSentiment指標を計算しない。

```javascript
// センチメントは長期（180秒以上）のみ
let sentimentResult = { sentiment: 'NEUTRAL', strength: 0 };
if (timeframeSeconds >= 180) {
  sentimentResult = this.sentiment.analyze(ticks, scaleFactor, periodScaleFactor);
}
```
