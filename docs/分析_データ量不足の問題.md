# 分析結果が出ない原因：データ量不足

## 🔍 現在の問題

**症状**:
- 上昇や下降のシグナルがほとんど出ない
- ほとんどがNEUTRAL判定（信頼度30-40%）
- テクニカル分析が機能していないように見える

**原因**: **分析に使うデータが少なすぎる**

---

## 📊 現在の分析データ量

### 15秒取引の場合

**データ収集**:
```javascript
// priceHistory に価格を蓄積（1秒ごと）
priceHistory.push(price);
if (priceHistory.length > 300) priceHistory.shift();
// 最大300秒（5分）分のデータを保持
```

**分析に使うデータ**:
```javascript
if (timeframeSeconds <= 15) {
  // 15秒取引: 直近30秒のデータのみ使用
  relevantPrices = prices.slice(-30);  // 30個のデータポイント
  relevantCandles = candles.slice(-30);
  relevantTicks = ticks.slice(-30);
}
```

**問題点**:
```
15秒取引で30秒分のデータ = たった30個のデータポイント

テクニカル指標の計算:
- MACD: 短期EMA(12), 長期EMA(26), シグナル(9) が必要
  → 最低でも26個必要だが、30個では不十分
- ADX: 14期間のDI計算が必要
  → 30個では正確な計算ができない
- Stochastic: 14期間が必要
  → 30個では不足
```

### データ量の比較

| 取引時間 | 使用データ範囲 | データポイント数 | 十分か？ |
|---------|--------------|----------------|---------|
| **15秒** | 直近30秒 | **30個** | ❌ 不足 |
| **30秒** | 直近60秒 | **60個** | ⚠️ ギリギリ |
| **60秒** | 直近120秒 | **120個** | ✅ やや十分 |
| **3分** | 直近300秒 | **300個** | ✅ 十分 |
| **5分** | 全データ | **300個** | ✅ 十分 |

## 🎯 各テクニカル指標が必要とするデータ数

### 1. MACD（Moving Average Convergence Divergence）

**計算に必要なデータ**:
- 短期EMA: 12期間
- 長期EMA: 26期間
- シグナルライン: 9期間

**必要最低データ数**: 26 + 9 = **35個**
**推奨データ数**: **50-100個**

**15秒取引（30個）の問題**:
```
データ: 30個
必要: 35個以上

→ 計算できるが、精度が非常に低い
→ EMAが安定しない
→ ノイズに敏感すぎる
```

### 2. ADX（Average Directional Index）

**計算に必要なデータ**:
- +DI, -DI: 14期間
- ADX: 14期間のDIの平滑化

**必要最低データ数**: 14 + 14 = **28個**
**推奨データ数**: **50-100個**

### 3. Stochastic

**計算に必要なデータ**:
- %K: 14期間
- %D: 3期間の%Kの平滑化

**必要最低データ数**: 14 + 3 = **17個**
**推奨データ数**: **30-50個**

### 4. RSI（Relative Strength Index）

**計算に必要なデータ**:
- 14期間の平均上昇/下降

**必要最低データ数**: **14個**
**推奨データ数**: **30-50個**

### 5. ATR（Average True Range）

**計算に必要なデータ**:
- 14期間のTrue Range

**必要最低データ数**: **14個**
**推奨データ数**: **30-50個**

### 6. ROC（Rate of Change）

**計算に必要なデータ**:
- 12期間の価格変化率

**必要最低データ数**: **12個**
**推奨データ数**: **20-30個**

## 📈 データ量と精度の関係

### 15秒取引（30個）の場合

```
MACD: 計算可能だが精度低い（推奨の30%）
ADX:  ギリギリ計算可能（推奨の30%）
Stochastic: 計算可能（推奨の60%）
RSI:  計算可能（推奨の50%）
ATR:  計算可能（推奨の50%）
ROC:  計算可能（推奨の100%）

総合: 精度が非常に低い
→ 各指標がバラバラの結果を出す
→ normalizedScore が -30 ~ +30 に収まる
→ NEUTRAL判定（信頼度30-40%）
```

### 60秒取引（120個）の場合

```
MACD: 精度高い（推奨の120%）
ADX:  精度高い（推奨の120%）
Stochastic: 精度非常に高い（推奨の240%）
RSI:  精度非常に高い（推奨の240%）
ATR:  精度非常に高い（推奨の240%）
ROC:  精度非常に高い（推奨の400%）

総合: 精度が高い
→ 各指標が一貫した結果を出す
→ normalizedScore が明確な値になる
→ HIGH/LOW判定（信頼度60-80%）
```

## 💡 解決策

### 解決策1: データ量を増やす（推奨）

**15秒取引の場合**:
```javascript
// 修正前
if (timeframeSeconds <= 15) {
  relevantPrices = prices.slice(-30);  // 30秒分
}

// 修正後
if (timeframeSeconds <= 15) {
  relevantPrices = prices.slice(-120);  // 120秒分（2分）
  // 15秒取引でも、2分間のトレンドを見て判断
}
```

**理由**:
- 15秒の取引でも、直近2分のトレンドを見る方が合理的
- テクニカル指標が正確に計算できる
- ノイズに強くなる

**修正後のデータ量**:

| 取引時間 | 使用データ範囲 | データポイント数 | 精度 |
|---------|--------------|----------------|------|
| **15秒** | 直近120秒（2分） | **120個** | ✅ 高い |
| **30秒** | 直近180秒（3分） | **180個** | ✅ 高い |
| **60秒** | 直近240秒（4分） | **240個** | ✅ 非常に高い |
| **3分** | 直近300秒（5分） | **300個** | ✅ 非常に高い |
| **5分** | 全データ | **300個** | ✅ 非常に高い |

### 解決策2: 判定基準の調整

データ量を増やした上で、判定基準も調整：

```javascript
// 15秒取引の場合、より敏感に反応
if (timeframeSeconds <= 15) {
  if (normalizedScore > 15) {  // 30 → 15 に緩和
    signal = 'HIGH';
    confidence = Math.min(85, 55 + normalizedScore / 2);
  }
}
```

### 解決策3: 複合的アプローチ

```javascript
if (timeframeSeconds <= 15) {
  // 1. データ量を増やす
  relevantPrices = prices.slice(-120);  // 2分

  // 2. 判定基準を緩和
  threshold = 15;  // 30 → 15

  // 3. NEUTRAL時の信頼度を上げる
  if (signal === 'NEUTRAL') {
    confidence = 50 + Math.abs(normalizedScore);  // 50-65%
  }
}
```

## 🎯 推奨する修正

### 修正内容

**multi-indicator-system.js の analyzeTimeframe 関数**:

```javascript
// 修正前
if (timeframeSeconds <= 15) {
  relevantPrices = prices.slice(-30);   // 30秒分
  relevantCandles = candles.slice(-30);
  relevantTicks = ticks.slice(-30);

// 修正後
if (timeframeSeconds <= 15) {
  // 15秒取引でも2分間のデータでトレンド判断
  relevantPrices = prices.slice(-120);   // 120秒分（2分）
  relevantCandles = candles.slice(-120);
  relevantTicks = ticks.slice(-120);

  // 30秒取引は3分間のデータ
  relevantPrices = prices.slice(-180);   // 180秒分（3分）
  relevantCandles = candles.slice(-180);
  relevantTicks = ticks.slice(-180);
}
else if (timeframeSeconds <= 60) {
  // 60秒取引は4分間のデータ
  relevantPrices = prices.slice(-240);   // 240秒分（4分）
  relevantCandles = candles.slice(-240);
  relevantTicks = ticks.slice(-240);
}
```

**判定基準も緩和**:

```javascript
// 修正後
if (normalizedScore > 50 && trendConfidence > 7) {
  signal = 'HIGH';
  confidence = Math.min(95, 70 + normalizedScore / 3);
}
else if (normalizedScore > 15) {  // 30 → 15 に緩和
  signal = 'HIGH';
  confidence = Math.min(85, 60 + normalizedScore / 2.5);
}
else if (normalizedScore < -50 && trendConfidence > 7) {
  signal = 'LOW';
  confidence = Math.min(95, 70 + Math.abs(normalizedScore) / 3);
}
else if (normalizedScore < -15) {  // -30 → -15 に緩和
  signal = 'LOW';
  confidence = Math.min(85, 60 + Math.abs(normalizedScore) / 2.5);
}
else {
  // NEUTRAL: -15 ~ +15
  signal = 'NEUTRAL';
  confidence = 50 + Math.abs(normalizedScore) * 1.5;  // 50-72%
}
```

## ❓ 実装しますか？

以下の修正を実装しますか？

1. **データ量の増加**:
   - 15秒: 30秒 → 120秒（2分）
   - 30秒: 60秒 → 180秒（3分）
   - 60秒: 120秒 → 240秒（4分）

2. **判定基準の緩和**:
   - HIGH判定: normalizedScore > 30 → > 15
   - LOW判定: normalizedScore < -30 → < -15
   - NEUTRAL範囲: -30~+30 → -15~+15

3. **NEUTRAL時の信頼度向上**:
   - 30 + |score|/2 → 50 + |score|*1.5
   - 30-45% → 50-72%

これにより、テクニカル分析が実用的な信頼度で動作するようになります。
