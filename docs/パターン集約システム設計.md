# パターン集約システム設計：全要素のパターン化

## 現在の類似度計算の全要素

### スコア配分（100点満点）

```javascript
類似度 =
  価格セグメント比較（40点）+
  テクニカル指標比較（30点）+
  その他（30点）
```

### 1. 価格セグメント比較（40点）

**現在の計算**:
```javascript
6セグメントの方向パターン:
現在: "UP-UP-UP-DOWN-DOWN-DOWN"
過去: "UP-UP-UP-DOWN-DOWN-DOWN"
→ 完全一致 = 40点
```

**パターン化の方法**:
```javascript
// セグメントパターンをキーにする
パターンキー: "UP-UP-UP-DOWN-DOWN-DOWN"

// 可能なパターン数
3^6 = 729パターン（UP/DOWN/FLAT の6セグメント）

実際はもっと少ない:
- 全部FLAT → 除外済み
- よくあるパターン: 50-100種類
- レアなパターン: 数百種類

推定: 200-300の有効パターン
```

✅ **パターン化可能！** セグメントパターンは既に離散値

---

### 2. テクニカル指標比較（30点）

**RSI比較（6点）**:

現在の計算:
```javascript
compareTechIndicator(currentRSI, pastRSI, 6点)
- トレンド方向一致: 40%
- 変化速度の類似度: 30%
- ボラティリティの類似度: 20%
- 強度の類似度: 10%
```

パターン化の方法:
```javascript
// RSIの状態を離散化
RSI値 → 状態

// 案A: 3段階（シンプル）
0-30:   "売られすぎ"
30-70:  "中立"
70-100: "買われすぎ"

// 案B: 5段階（バランス）
0-20:   "超売られすぎ"
20-40:  "売られすぎ"
40-60:  "中立"
60-80:  "買われすぎ"
80-100: "超買われすぎ"

// トレンド方向
UP / DOWN / NEUTRAL

// 総合パターン
RSIパターン = "値:買われすぎ|トレンド:UP|速度:強"
```

**MACD比較（6点）**:

```javascript
// MACDの状態を離散化
MACD強度 → 状態

// 3段階
< -0.02: "強い下降"
-0.02〜0.02: "横ばい"
> 0.02: "強い上昇"

// トレンド
UP / DOWN / NEUTRAL

// 総合パターン
MACDパターン = "強度:強い上昇|トレンド:UP"
```

**ROC、Stochastic、ADX も同様**

## パターンキーの設計

### 方法1: 完全分類（最高精度、最重）

```javascript
パターンキー =
  セグメント +
  RSI詳細 +
  MACD詳細 +
  ROC詳細 +
  Stochastic詳細 +
  ADX詳細 +
  MA Cross詳細

例:
"SEG:UP-UP-UP-DOWN-DOWN-DOWN|" +
"RSI:買われすぎ-UP-強|" +
"MACD:上昇-UP-中|" +
"ROC:上昇-UP-弱|" +
"STOCH:買われすぎ-DOWN-弱|" +
"ADX:強いトレンド|" +
"MACROSS:ゴールデンクロス-UP"

パターン数の推定:
300（セグメント）×
27（RSI: 3値×3トレンド×3速度）×
27（MACD）×
27（ROC）×
27（STOCH）×
9（ADX: 3値×3トレンド）×
9（MACROSS: 3状態×3トレンド）
= 300 × 27^4 × 9^2
= 1,432,890,000 パターン ❌

→ パターン数が爆発
→ 各パターンに0.000004件しかない
→ 統計的に無意味
```

---

### 方法2: 主要要素のみ（バランス）

```javascript
パターンキー =
  セグメント（完全）+
  RSI（3段階）+
  MACD（3段階）+
  トレンド強度（3段階）

例:
"SEG:UP-UP-UP-DOWN-DOWN-DOWN|" +
"RSI:高|" +
"MACD:上昇|" +
"TREND:強"

パターン数の推定:
300（セグメント）×
3（RSI）×
3（MACD）×
3（トレンド強度）
= 8,100 パターン

各パターンのデータ数（6000件の場合）:
6000 / 8100 = 0.74件 ❌

→ まだデータ不足
```

---

### 方法3: 2層構造（推奨）

```javascript
// 第1層: 粗いパターン分類
粗いキー =
  セグメント（簡略: 主要パターンのみ）+
  RSI（3段階）+
  MACD（3段階）

例:
"SEG:強い上昇|RSI:高|MACD:上昇"

セグメント簡略化:
- "強い上昇": UP-UP-UP-* のような
- "強い下降": DOWN-DOWN-DOWN-* のような
- "反転上昇": DOWN-DOWN-UP-UP-UP-UP
- "反転下降": UP-UP-DOWN-DOWN-DOWN-DOWN
- "レンジ": UP-DOWN-UP-DOWN のような
→ 10-20の主要パターン

パターン数:
20（セグメント簡略）× 3（RSI）× 3（MACD）
= 180 パターン

各パターンのデータ数:
6000 / 180 = 33件 ✅ 統計的に意味がある

// 第2層: パターン内で詳細計算
パターンAのデータ（33件）の中で:
→ 詳細な類似度計算（全指標使用）
→ 上位20件で予測

処理時間:
第1層検索: 0.01秒
第2層計算（33件）: 0.02秒
合計: 0.03秒（98%削減） ✅
```

## 実装案の比較

### 案A: 完全パターン集約（超シンプル、精度低下）

```javascript
// パターンごとに統計値だけ保存
patterns = {
  "SEG:UP-UP-UP-DOWN-DOWN-DOWN|RSI:高|MACD:上昇": {
    count: 100,
    highCount: 70,
    lowCount: 30,
    avgChange: 0.05,
    // 詳細データは保存しない
  }
}

// 分析時
現在の相場 → パターンキー生成
→ patterns[key] から統計値取得
→ HIGH 70%と即座に予測

処理時間: 0.001秒（99.9%削減） ✅
精度: 70-80%（推定、20-30%低下） ⚠️
```

**メリット**:
- 超高速
- 超省メモリ
- 実装がシンプル

**デメリット**:
- 詳細な類似度計算ができない
- 精度低下が不明（実測必要）
- 統計値だけでは不十分な可能性

### 案B: 2層ハイブリッド（推奨）

```javascript
// 粗いパターンでグループ化
patternGroups = {
  "SEG:強い上昇|RSI:高|MACD:上昇": [
    data1, data2, ..., data33  // 詳細データを保持
  ]
}

// 分析時
現在の相場 → 粗いパターンキー生成
→ patternGroups[key] のデータを取得（33件）
→ この33件だけで詳細な類似度計算
→ 上位20件で予測

処理時間: 0.03秒（98%削減） ✅
精度: 95-100%（ほぼ維持） ✅
```

**メリット**:
- 大幅な高速化
- 精度をほぼ維持
- バランスが良い

**デメリット**:
- やや複雑
- パターン設計が重要

### 案C: インデックス方式（最高精度、中程度の高速化）

```javascript
// 各要素にインデックスを作成
indexes = {
  segment: {
    "UP-UP-UP-DOWN-DOWN-DOWN": [data1, data3, data7, ...],
    "DOWN-DOWN-UP-UP-UP-UP": [data2, data5, data8, ...]
  },
  rsi: {
    "高": [data1, data2, data9, ...],
    "中": [data3, data4, data10, ...]
  },
  macd: {
    "上昇": [data1, data5, data11, ...],
    "下降": [data2, data6, data12, ...]
  }
}

// 分析時
現在のセグメント → candidates1 = indexes.segment[key]（500件）
現在のRSI → candidates2 = indexes.rsi[key]（2000件）
candidates = candidates1 ∩ candidates2（100件）← 積集合

→ この100件で詳細計算

処理時間: 0.1秒（95%削減） ✅
精度: 100%（完全維持） ✅
```

**メリット**:
- 精度完全維持
- 柔軟（条件追加が簡単）
- 理解しやすい

**デメリット**:
- メモリ使用量が多い
- やや複雑

## パターン簡略化の詳細設計

### セグメントパターンの簡略化

```javascript
// 完全パターン → 簡略パターン
function simplifySegmentPattern(segments) {
  const pattern = segments.map(s => s.direction).join('-');

  // 強い上昇トレンド
  if (pattern.match(/^(UP-){4,6}/)) return "強い上昇";

  // 強い下降トレンド
  if (pattern.match(/^(DOWN-){4,6}/)) return "強い下降";

  // 上昇反転
  if (pattern.match(/DOWN.*UP-UP-UP/)) return "上昇反転";

  // 下降反転
  if (pattern.match(/UP.*DOWN-DOWN-DOWN/)) return "下降反転";

  // レンジ相場
  if (pattern.match(/(UP-DOWN|DOWN-UP){2,}/)) return "レンジ";

  // その他（上昇寄り/下降寄り/横ばい）
  const upCount = segments.filter(s => s.direction === 'UP').length;
  const downCount = segments.filter(s => s.direction === 'DOWN').length;

  if (upCount > downCount + 2) return "上昇寄り";
  if (downCount > upCount + 2) return "下降寄り";
  return "横ばい";
}

// 結果: 10パターン
主要パターン:
1. 強い上昇
2. 強い下降
3. 上昇反転
4. 下降反転
5. レンジ
6. 上昇寄り
7. 下降寄り
8. 横ばい
9. 複雑（その他）
```

function classifyRSI(rsiValue, rsiTrend) {

function classifyMACD(macdStrength, macdTrend) {

}
