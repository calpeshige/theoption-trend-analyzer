# スマホ連携（PC→スマホ シグナル転送）セットアップ

PCのChrome拡張が生成するシグナルを、スマホのPWAでリアルタイム確認＋通知する機能のセットアップ手順。

## 構成概要

```
PC拡張 (theoption-analyzer.js → mobile-relay.js)
  └ Firestore live_signals/{PC版ライセンスキー} へ書き込み（変化時＋30秒ハートビート）
        │ onSnapshot（変化時のみ）
  スマホPWA (mobile/) ◀┘  entryAtからローカルでカウントダウン
```

- 既存のFirebaseプロジェクト `theoption-license` をそのまま再利用。
- スマホ版は **MOB- ライセンス**で認証（`license-admin.html` で発行・PC版とペアリング）。
- PWAは単一コードでAndroid/iPhone両対応。Vercelの同一プロジェクトに `mobile/` をデプロイ。

## 1. Firestore セキュリティルール（必須）

`live_signals` コレクションへの読み書きを許可する必要があります。Firebase Console → Firestore Database → ルール に以下を追加してください（既存の `licenses` ルールは残したまま追記）。

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // 既存: ライセンス（読み取り＋devicesの更新を許可）
    match /licenses/{key} {
      allow read: if true;
      allow update: if true;   // ※現状の運用に合わせる（devices登録のため）
    }

    // 追加: ライブシグナル中継
    match /live_signals/{channel} {
      allow read: if true;     // スマホPWAが購読
      allow write: if true;    // PC拡張が書き込み（docId=推測困難なPC版キー）
    }
  }
}
```

> 注: 現状の設計はAPIキーのみ（Firebase Auth未使用）。`docId` がランダムなPC版ライセンスキーであること自体が軽い秘匿になっています。より厳格にする場合は Anonymous Auth 導入（将来のP3）を検討してください。

## 2. PWAのデプロイ

`vercel.json` に `mobile/**` のビルドと `/mobile/` ルートを追加済み。通常のVercelデプロイで `https://<ドメイン>/mobile/` に公開されます。

- 公開URL例: `https://<ドメイン>/mobile/?ch=<PC版ライセンスキー>`
- **重要**: 拡張側の公開URLは `sidepanel.js` の定数 `MOBILE_APP_BASE_URL` で指定しています。実際のVercelドメイン（license-admin.html と同じドメイン）に合わせて確認・修正してください。

```js
// sidepanel.js
const MOBILE_APP_BASE_URL = 'https://theoptiontrend.vercel.app';
```

## 3. 使い方（エンドユーザー）

1. PC拡張のサイドパネル → 設定 → 「📱 スマホ連携」をONにする。
2. 表示されたQRコードをスマホで読み取る（またはURLをコピーして開く）。
3. スマホでスマホ版ライセンスキー（MOB-）を入力して認証。
4. PCのシグナルがリアルタイム表示され、新規シグナルで音／バイブ通知。
5. **iPhoneの場合**: 画面を閉じても通知を受け取るには「共有 → ホーム画面に追加」でアプリ化が必要（iOS 16.4+。背面プッシュはP2で対応）。

## 4. 動作確認（P0/P1）

- **P0**: PCでシグナル発生時、Firebase Console で `live_signals/{PC版キー}` が「変化時＋約30秒ごと」に更新されること。毎秒は書かれないこと。
- **P1**: スマホでMOB認証 → シグナルがリアルタイム表示、カウントダウンがローカルで進む、PC停止で「PCオフライン」表示、新規シグナルで音/バイブ。無効/期限切れ/台数超過のMOBキーが弾かれること。

## 5. 今後（P2: 背面プッシュ）

画面を閉じた状態でも通知を受けるには Web Push/FCM を追加します（別途実装）。
- Firebase Console で Cloud Messaging API (V1) 有効化
- サービスアカウント鍵 → Vercel環境変数 `FIREBASE_SERVICE_ACCOUNT`
- Web Push証明書(VAPID) → PWAに設定
- `mobile/firebase-messaging-sw.js` と Vercelサーバーレス関数 `mobile/api/push.js` を追加
- PCは新規シグナル時のみ push API を叩く
