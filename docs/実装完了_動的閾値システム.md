# 動的閾値システム実装完了

## 実装日時
2025-11-02

## 問題点

**固定閾値の問題**:
- 全通貨ペアに対して固定の閾値 `0.02%` を使用
- 仮想通貨（BTC/USD）: ほとんど全てUP/DOWNになる（閾値が低すぎる）
- 低ボラティリティ通貨（AUD/NZD）: ほとんど全てFLATになる（閾値が高すぎる）

**結果**:
- セグメントハッシュがほとんど「NNNNNN」（全FLAT）になる
- パターンマッチングが機能しない

## 解決策

### **ATRベース + 統計ベースの動的閾値システム**

各通貨ペアの現在のボラティリティに応じて、自動的に最適な閾値を計算します。

```javascript
calculateDynamicThreshold(priceHistory, atrPercent) {
  // 方法1: ATRベースの閾値
  // - ATRの10%をノイズレベルとする
  atrBasedThreshold = atrPercent * 0.1;

  // 方法2: 統計ベースの閾値
  // - 過去100件の1秒ごとの変化率の標準偏差を計算
  // - 標準偏差の25%をノイズレベルとする
  statsBasedThreshold = stdDev * 0.25;

  // 加重平均で統合
  threshold = atrBasedThreshold * 0.6 + statsBasedThreshold * 0.4;

  return threshold;
}
```

## 動作の仕組み

### 閾値計算の流れ

```
1. ATRベース閾値計算
   └─ ATR% * 0.1 → 現在の市場ボラティリティを反映

2. 統計ベース閾値計算
   └─ 過去100件の変化率 → 標準偏差 * 0.25 → その通貨ペアの典型的なノイズレベル

3. 加重平均で統合
   └─ (ATR閾値 * 0.6) + (統計閾値 * 0.4) → 最終閾値

4. 範囲制限
   └─ 0.001% ≤ 閾値 ≤ 0.1%
```

### 通貨ペアごとの動的調整（例）

| 通貨ペア | ATR% | 統計StdDev | 最終閾値 | 効果 |
|---------|------|------------|---------|------|
| BTC/USD | 0.15% | 0.08% | 0.017% | 固定0.02%より少し緩い |
| EUR/USD | 0.05% | 0.02% | 0.008% | 固定0.02%より厳しい |
| AUD/NZD | 0.02% | 0.01% | 0.003% | 固定0.02%より大幅に緩い ✅ |

## 期待される効果

### ✅ 改善点

1. **通貨ペア別最適化**
   - 仮想通貨: 大きな動きのみ検出（ノイズ除外）
   - マイナー通貨: 小さな動きも検出（FLAT過剰判定の解消）

2. **市場環境への適応**
   - 高ボラティリティ時: 閾値自動上昇
   - 低ボラティリティ時: 閾値自動低下

3. **パターンマッチング精度向上**
   - FLAT判定が適切になる → セグメントハッシュが意味を持つ
   - 「NNNNNN」ばかりにならない → パターンの多様性が確保される

## 注意事項

- キャッシュ有効期限: 1分間
  - 同じ通貨ペアで1分以内なら再計算しない（パフォーマンス最適化）
- 最初の100件データが集まるまでは統計ベース閾値が不正確
  - その場合はATRベースが主導的になる
