# エントリー条件のロジック詳細

## 概要

エントリー条件は**テクニカル分析4項目**と**AI予測4項目**の合計**8項目**で構成されています。

- 各項目は **✓（条件クリア）** または **✗（条件未クリア）** で表示されます
- このツールが「エントリー推奨」「エントリー非推奨」といった自動判定を行うことはありません。最終的な判断は常にユーザー自身に委ねられています

---

## テクニカル分析（左列）の4項目

### 1. トレンド方向

| 項目 | 内容 |
|------|------|
| **表示名** | トレンド方向 |
| **何を確認しているか** | 今の相場の流れ（上昇傾向か下降傾向か）と、シグナルの向きが合っているかどうか |
| **✓になる条件** | HIGHシグナルが出た時に相場が上昇傾向、LOWシグナルが出た時に相場が下降傾向であれば✓ |
| **✗になる条件** | 相場が下落中なのにHIGHが出ている、横ばい（レンジ）の時にシグナルが出ているなど、方向が合わない場合は✗ |

#### 判定の仕組み

相場の状態は「レジーム」と呼ばれる9つの分類で判定されます。

**上昇系（HIGHシグナルと一致）**
- 強い上昇トレンド
- 弱い上昇トレンド
- 上方ブレイクアウト
- 上方反転（下落から上昇への転換）

**下降系（LOWシグナルと一致）**
- 強い下降トレンド
- 弱い下降トレンド
- 下方ブレイクアウト
- 下方反転（上昇から下落への転換）

**中立**
- レンジ（横ばい）

#### レジーム判定の基準

レジームの判定には主に**ADX（トレンドの強さを測る指標）**と**EMA（指数移動平均線）**を使用しています。

| 条件 | 判定結果 |
|------|----------|
| ADXが28以上 | 強いトレンド |
| ADXが18以上 | 弱いトレンド |
| 急激な値動き幅の拡大 + ボリンジャーバンド外側 | ブレイクアウト |
| RSIが極端な位置 + ボリンジャーバンド端 | 反転 |
| どれにも当てはまらない | レンジ |

なお、ボラティリティ（相場の変動の大きさ）に応じてADXの閾値は動的に変化します（低ボラ時は-3、高ボラ時は+3から+5）。

#### 現状の問題

- シグナル（HIGH/LOW）は5つの時間軸の7つの指標の多数決で決まりますが、レジームはADXとEMAという**別のロジック**で判定されます。この2つは独立しているため、シグナルがHIGHでもレジームが「レンジ」と判定されるケースが頻発します
- 超短期取引（15秒から5分）ではADXが18に達しにくく、レジームが「レンジ」のままになることが多いです。レンジはどちらの方向にも一致しないため、常に✗になります
- **結果として、この条件が✓になることはかなり稀です**

### 2. 値動きの安定性

| 項目 | 内容 |
|------|------|
| **表示名** | 値動きの安定性 |
| **何を確認しているか** | 直近の値動きの大きさが普段と比べて異常でないか |
| **✓になる条件** | 直近の値動き幅が過去の平均と比べて0.6倍から1.4倍の範囲（通常）であれば✓ |
| **✗になる条件** | 0.6倍未満（動きが小さすぎる）、1.4倍から2.0倍（高ボラティリティ）、2.0倍超（急変動）の場合は✗ |

#### 判定の仕組み

ATR（Average True Range）という、一定期間の値動き幅の平均を表す指標を使います。

1. 直近14期間のATRを計算する
2. ATRの履歴を最大100期間保持し、過去50期間の平均ATRを算出する
3. 「現在のATR / 平均ATR」= ボラティリティ比率で判定する

| ボラティリティ比率 | 状態 | 判定 |
|:------------------:|------|:----:|
| 0.6未満 | 値動きが小さすぎる | ✗ |
| 0.6 から 1.4 | 通常 | ✓ |
| 1.4 から 2.0 | 高ボラティリティ | ✗ |
| 2.0超 | 急変動 | ✗ |

#### 現状の評価

相場の大半の時間帯ではボラティリティ比率が0.6から1.4に収まるため、通常は✓になります。**妥当な閾値設定です。**

---

### 3. 時間軸の一致

| 項目 | 内容 |
|------|------|
| **表示名** | 時間軸の一致 |
| **何を確認しているか** | 短い時間軸から長い時間軸まで、同じ方向を向いているかどうか |
| **✓になる条件** | 5つの時間軸のうち70%以上（4つ以上）が同じ方向であれば✓ |
| **✗になる条件** | 3つ以下しか同じ方向でない場合は✗ |

#### 判定の仕組み

5つの時間軸（15秒・30秒・60秒・180秒・300秒）それぞれでテクニカル分析を行います。各時間軸では以下の7つの指標をもとにHIGH/LOW/NEUTRAL（上昇/下落/中立）を判定します。

- RSI
- MACD
- 移動平均クロスオーバー
- ボリンジャーバンド
- ADX
- ストキャスティクス
- モメンタム

5つの時間軸のうち、最も多い方向の割合で✓/✗が決まります。

| 一致数 | 一致率 | 判定 |
|:------:|:------:|:----:|
| 5つ中5つ | 100% | ✓ |
| 5つ中4つ | 80% | ✓ |
| 5つ中3つ | 60% | ✗ |
| 5つ中2つ以下 | 40%以下 | ✗ |

#### 現状の問題

- 70%以上、つまり5つ中4つ以上の一致が必要です。超短期取引では15秒足と300秒足で方向が異なるのはよくあることで、4/5一致はかなりハードルが高いです
- 3/5一致（60%）でもある程度の方向性は確認できますが、現在は✗と判定されます
- **結果として、この条件が✓になることはかなり稀です**

---

### 4. 初動の検出

| 項目 | 内容 |
|------|------|
| **表示名** | 初動の検出 |
| **何を確認しているか** | トレンドの始まりを捉えられているかどうか |
| **✓になる条件** | RSI・ストキャスティクス・MACDの3指標が示す初動のスコアが50以上で、かつシグナルと同じ方向であれば✓ |
| **✗になる条件** | スコアが50未満、方向が不一致、指標が極端な位置（買われすぎ・売られすぎ）にある場合は✗ |

#### 判定の仕組み

初動検出は以下の4つの要素のスコアを合算して判定します。

**a) RSI初動**

RSIの中立ゾーン（45から55）と極端ゾーン（25以下 or 75以上）を基準に判定します。

| 状態 | スコア |
|------|--------|
| 中立ゾーンから上に離脱 | (現在値 - 55) x 10 点 |
| 中立ゾーンから下に離脱 | (45 - 現在値) x 10 点 |
| 中立ゾーン内で2pt以上の動き | 30点 |
| 中立ゾーン外で継続中 | 50点 |
| 極端ゾーン到達（買われすぎ/売られすぎ） | 0点（全体が「見送り」になる） |

**b) ストキャスティクス初動**

ストキャスティクスの中立ゾーン（40から60）と極端ゾーン（15以下 or 85以上）を基準に、RSI初動と同様のロジックで判定します。

**c) RSIモメンタム（勢いの変化）**

| 状態 | スコア |
|------|--------|
| 上昇加速中（変化量 > 0.5、加速度 > 0.3） | +30点 |
| 下落加速中 | +30点 |
| 上昇減速中 | 逆方向に+20点 |
| 下落減速中 | 逆方向に+20点 |

**d) MACDモメンタム（勢いの変化）**

| 状態 | スコア |
|------|--------|
| 上昇/下落加速中 | +25点 |
| 反転シグナル検出 | +40点 |

**最終判定**: HIGH側スコアとLOW側スコアを比較し、大きい方が50以上ならそのシグナルを採用します。シグナルと同方向なら✓になります。

#### 現状の問題

- RSI初動のスコアは、たとえばRSIが55から56に動いた程度では10点にしかなりません
- RSI初動単体ではスコア50に届かないため、モメンタム加速（+30）や反転シグナル（+40）の追加ポイントが必須です
- 指標の履歴は最大10期間しか保持されないため、タイミングがずれると検出できません
- **結果として、複数の条件が同時に揃わないと✓にならず、出現頻度はかなり低いです**

## AI予測（右列）の4項目

### 1. 予測の偏り

| 項目 | 内容 |
|------|------|
| **表示名** | 予測の偏り |
| **何を確認しているか** | AIの予測が上昇と下降のどちらかに明確に偏っているか |
| **✓になる条件** | 上昇確率と下降確率の差が20ポイント以上であれば✓ |
| **✗になる条件** | 差が20ポイント未満であれば✗ |

例: 上昇65% vs 下降40% → 差25pt → ✓
例: 上昇52% vs 下降45% → 差7pt → ✗

---

### 2. 引き分けリスク

| 項目 | 内容 |
|------|------|
| **表示名** | 引き分けリスク |
| **何を確認しているか** | 同値（引き分け）になる確率が低いかどうか |
| **✓になる条件** | 同値率が20%以下であれば✓ |
| **✗になる条件** | 同値率が20%超の場合は✗ |

同値率は「100% - 上昇確率 - 下降確率」で算出されます。

---

### 3. データ量

| 項目 | 内容 |
|------|------|
| **表示名** | データ量 |
| **何を確認しているか** | 予測の根拠となる学習データが十分にあるか |
| **✓になる条件** | 類似パターンのマッチ件数が100件以上であれば✓ |
| **✗になる条件** | 100件未満の場合は✗ |

---

### 4. 方向の一致

| 項目 | 内容 |
|------|------|
| **表示名** | 方向の一致 |
| **何を確認しているか** | AI予測の方向とテクニカル分析のシグナル方向が合っているか |
| **✓になる条件** | AI予測が上昇優位でテクニカルがHIGH、またはAI予測が下降優位でテクニカルがLOWであれば✓ |
| **✗になる条件** | AIとテクニカルが逆方向の場合は✗ |

## 改善の方向性（検討中）

### トレンド方向について

レジーム（ADX+EMA）とシグナル（7指標多数決）が別系統なので不一致が起きやすいという根本的な問題があります。

- **案1**: 判定ロジックを見直す（例: レジームではなく複数時間軸の方向とシグナル方向で比較する）。ただし、常に一致してしまい条件として意味がなくなる可能性がある
- **案2**: 別の条件に差し替える

### 時間軸の一致について

- 閾値を70%から**60%に緩和**する（5つ中3つ一致でも✓にする）

### 初動の検出について

- スコア閾値を50から**30に緩和**する
- または、NEUTRAL以外（何らかの方向性が検出された）なら✓にする
