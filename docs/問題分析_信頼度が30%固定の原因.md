# 問題分析：信頼度が30%台で固定される原因

## 🎯 問題の本質

**症状**: テクニカル分析の信頼度が更新しても30%台しか表示されない

**根本原因**: 超短期取引（15秒～5分）の価格変動が小さすぎて、全ての指標が「ほぼ0」の強度値を返している

---

## 📊 各指標の問題点

### 1. MACD Indicator（line 38）

**コード**:
```javascript
const strength = Math.max(-10, Math.min(10, histogram * 100));
```

**問題**:
- `histogram`は通常0.0001～0.001程度（超短期では価格変動が小さい）
- `0.001 * 100 = 0.1` → strength = 0.1
- **実際の出力**: ほぼ0

**想定**: ±10の範囲でスコアを返す
**現実**: ±0.1程度しか返せていない

---

### 2. ADX Indicator（line 75）

**コード**:
```javascript
const strength = Math.min(10, adx / 5);
```

**問題**:
- ADXは0～100の範囲だが、超短期では明確なトレンドが形成されにくい
- 通常のADX値: 10～20程度（弱いトレンド）
- `15 / 5 = 3` → strength = 3
- **実際の出力**: 2～4程度

**想定**: トレンドの強さを0～10で評価
**現実**: 超短期ではトレンドが弱く、常に低い値

---

### 3. Stochastic Indicator（lines 103-115）

**コード**:
```javascript
if (k > 80 && d > 80) {
  strength = -5;  // 買われすぎ
} else if (k < 20 && d < 20) {
  strength = -5;  // 売られすぎ
} else if (k > d && k < 80) {
  strength = 5;   // 上昇トレンド
} else if (k < d && k > 20) {
  strength = -5;  // 下降トレンド
} else {
  strength = 0;   // その他
}
```

**問題**:
- 条件が厳しすぎる（k > 80 AND d > 80など）
- 超短期では両方の条件を同時に満たすことが稀
- **実際の出力**: ほとんど0、たまに±5

---

### 4. ATR Indicator（lines 151-162）

**コード**:
```javascript
const atrPercent = (atr / currentPrice) * 100;

if (atrPercent > 0.5) {
  strength = 3;  // HIGH volatility
} else if (atrPercent > 0.2) {
  strength = 2;  // MODERATE volatility
} else {
  strength = 1;  // LOW volatility
}
```

**問題**:
- FX通貨ペアの超短期ATR%: 通常0.01%～0.1%程度
- `0.05% < 0.2%` → strength = 1（常に最低値）
- **実際の出力**: ほぼ常に1

**想定**: ボラティリティを1～3で評価
**現実**: 超短期では常に1

---

### 5. ROC Indicator（lines 185-200）

**コード**:
```javascript
const roc = ((currentPrice - pastPrice) / pastPrice) * 100;

if (roc > 1.0) {
  strength = 10;       // 1%以上の上昇
} else if (roc > 0.3) {
  strength = 5;        // 0.3%以上の上昇
} else if (roc < -1.0) {
  strength = -10;      // 1%以上の下降
} else if (roc < -0.3) {
  strength = -5;       // 0.3%以上の下降
} else {
  strength = 0;        // それ以外
}
```

**問題**:
- 15秒取引の10秒間のROC: 通常±0.01%～±0.1%程度
- `0.05% < 0.3%` → strength = 0
- **実際の出力**: ほぼ常に0

**想定**: ±10の範囲でスコアを返す
**現実**: 超短期ではほとんど0

---

### 6. Market Sentiment Analyzer（lines 227-246）

**コード**:
```javascript
if (upRatio > 0.65) {
  strength = 5;
} else if (upRatio < 0.35) {
  strength = -5;
} else {
  strength = 0;
}

// 強度調整
if (avgChange > 10) {
  strength *= 1.5;
} else if (avgChange < 5) {
  strength *= 0.5;
}
```

**問題**:
- 超短期の`avgChange`: 通常1～3程度（価格が0.001単位でしか動かない）
- `avgChange = 2 < 5` → `strength = 5 * 0.5 = 2.5`
- **実際の出力**: ±2.5程度

**想定**: ±5～±7.5の範囲
**現実**: ±2.5程度（半減されている）

## 🔢 実際の計算例

### 典型的なケース（15秒取引）

**各指標の実際の出力**:
```javascript
{
  macd: 0.1,        // histogram * 100 ≈ 0
  adx: 3,           // ADX 15 → 15/5 = 3
  stochastic: 0,    // 条件不一致
  atr: 1,           // atrPercent < 0.2%
  roc: 0,           // ROC < 0.3%
  sentiment: 2.5    // base 5 * 0.5 = 2.5
}
```

**スコア計算**:
```javascript
totalScore =
  + macd * 2         = 0.1 * 2 = 0.2
  + stochastic * 1.5 = 0 * 1.5 = 0
  + roc * 2          = 0 * 2 = 0
  + sentiment * 2    = 2.5 * 2 = 5
  = 5.2

maxScore = 100

normalizedScore = (5.2 / 100) * 100 = 5.2
```

**判定**:
```javascript
// -15 < 5.2 < 15 → NEUTRAL
signal = 'NEUTRAL'
confidence = 50 + |5.2| * 1.5 = 50 + 7.8 = 57.8%
```

**結果**: 57%の信頼度で「見送り」

## 🤔 なぜ30%台なのか？

上記の計算では57%になるはずなのに、実際は30%台が表示されている。

**可能性1**: データ不足
- `prices.length < 26` → MACD = 0
- `candles.length < 14` → ADX = 0, Stochastic = 0, ATR = 0
- すべての指標が0を返す → totalScore = 0 → normalizedScore = 0
- confidence = 50 + 0 = 50%（これでも50%だが...）

**可能性2**: 別の計算ロジックが影響
- `theoption-analyzer.js`側で信頼度を再計算している？
- UIの表示ロジックに問題がある？

**可能性3**: キャッシュされた古い結果
- タブ切り替え時のキャッシュが古い値を保持している？

---

## ✅ 解決策

### 方法1: 超短期専用の閾値を使用（推奨）

各指標の判定基準を超短期取引用に調整：

**MACD**:
```javascript
// 変更前: histogram * 100
// 変更後: histogram * 10000（100倍）
const strength = Math.max(-10, Math.min(10, histogram * 10000));
```

**ROC**:
```javascript
// 変更前: roc > 1.0, roc > 0.3
// 変更後: roc > 0.1, roc > 0.03（1/10）
if (roc > 0.1) {
  strength = 10;
} else if (roc > 0.03) {
  strength = 5;
}
```

**ATR**:
```javascript
// 変更前: atrPercent > 0.5, > 0.2
// 変更後: atrPercent > 0.05, > 0.02（1/10）
if (atrPercent > 0.05) {
  strength = 3;
} else if (atrPercent > 0.02) {
  strength = 2;
} else {
  strength = 1;
}
```

**Sentiment**:
```javascript
// 変更前: avgChange > 10, > 5
// 変更後: avgChange > 1, > 0.5（1/10）
if (avgChange > 1) {
  strength *= 1.5;
} else if (avgChange < 0.5) {
  strength *= 0.5;
}
```

---

### 方法2: 相対的な強度評価（さらに推奨）

絶対値の閾値ではなく、相対的な変化率で評価：

**例: MACD**
```javascript
// 直近の平均histogramと比較
const avgHistogram = recentHistograms.average();
const relativeStrength = (histogram - avgHistogram) / Math.abs(avgHistogram);
const strength = Math.max(-10, Math.min(10, relativeStrength * 10));
```

**例: ROC**
```javascript
// 直近の平均ROCと比較
const avgROC = recentROCs.average();
const deviation = roc - avgROC;
const strength = Math.max(-10, Math.min(10, deviation * 100));
```

---

### 方法3: 時間枠別の閾値設定

`analyzeTimeframe()`に時間枠情報を渡して、動的に閾値を調整：

```javascript
analyzeTimeframe(data, timeframeSeconds) {
  const scaleFactor = this.getScaleFactor(timeframeSeconds);

  // MACD
  const macdStrength = histogram * (100 * scaleFactor);

  // ROC
  const rocThresholdHigh = 1.0 / scaleFactor;
  const rocThresholdLow = 0.3 / scaleFactor;
}

getScaleFactor(timeframeSeconds) {
  if (timeframeSeconds <= 15) return 100;    // 15秒: 100倍
  if (timeframeSeconds <= 30) return 50;     // 30秒: 50倍
  if (timeframeSeconds <= 60) return 20;     // 60秒: 20倍
  if (timeframeSeconds <= 180) return 5;     // 3分: 5倍
  return 1;                                   // 5分: 通常
}
```
