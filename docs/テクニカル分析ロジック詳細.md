# テクニカル分析ロジック詳細書

## 概要

本システムは、超短期バイナリーオプション取引（15秒〜5分）向けのテクニカル分析エンジンです。
10種類の指標を統合し、HIGH（上昇予測）/ LOW（下降予測）/ NEUTRAL（見送り）のシグナルを生成します。

**対象プラットフォーム**: TheOption
**対象時間枠**: 15秒、30秒、60秒、180秒（3分）、300秒（5分）

---

## 1. 使用指標一覧

### 1.1 MACD（Moving Average Convergence Divergence）

**目的**: トレンドの方向と勢いを測定

**計算方法**:
```
EMA12 = 12期間の指数移動平均
EMA26 = 26期間の指数移動平均
MACD = EMA12 - EMA26
Signal = MACD × 0.8（簡易版）
Histogram = MACD - Signal
Strength = Histogram × 100 × scaleFactor（-10〜+10に正規化）
```

**パラメータ調整**:
- 期間は `periodScaleFactor` でスケーリング
- 15秒: period12=3, period26=7（0.25倍）
- 60秒: period12=12, period26=26（基準）
- 300秒: period12=60, period26=130（5倍）

**重み**: 1.2〜1.8（時間枠により変動）

---

### 1.2 ADX（Average Directional Index）

**目的**: トレンドの強さを測定（方向は判定しない）

**計算方法**:
```
観測期間 = 判定時間 × 2（例: 15秒判定 → 30秒観測）
期間 = 30（固定）

+DM = 当日高値 - 前日高値（正の場合のみ）
-DM = 前日安値 - 当日安値（正の場合のみ）
TR = max(高値-安値, |高値-前日終値|, |安値-前日終値|)

+DI = (+DM / TR) × 100
-DI = (-DM / TR) × 100
ADX = |+DI - -DI| / (+DI + -DI) × 100

Strength = min(10, ADX / 5)
```

**特徴**:
- 全時間枠で一貫して30期間のADX計算
- 観測期間は判定時間の2倍（騙しシグナル対策）
- `trendConfidence = ADX Strength × 1.5`（シグナル判定に使用）

**重み**: 1.5（固定）

---

### 1.3 Stochastic Oscillator

**目的**: 買われすぎ・売られすぎの判定

**計算方法**:
```
期間 = 14 × periodScaleFactor

%K = (現在終値 - 期間内最安値) / (期間内最高値 - 期間内最安値) × 100
%D = %K × 0.9（簡易版）
```

**シグナル判定**:
| 条件 | シグナル | Strength |
|------|----------|----------|
| K > 80 かつ D > 80 | OVERBOUGHT（買われすぎ） | -5 |
| K < 20 かつ D < 20 | OVERSOLD（売られすぎ） | -5 |
| K > D かつ K < 80 | BULLISH | +5 |
| K < D かつ K > 20 | BEARISH | -5 |

**重み**: 1.5（固定）

---

### 1.4 ATR（Average True Range）

**目的**: ボラティリティ（価格変動幅）の測定

**計算方法**:
```
TR = max(高値-安値, |高値-前日終値|, |安値-前日終値|)
ATR = TR の期間平均（14期間）
ATRパーセント = (ATR / 現在価格) × 100
```

**ボラティリティ判定**:
| ATRパーセント | 判定 | Strength |
|---------------|------|----------|
| > 0.5% / scaleFactor | HIGH | 3 |
| > 0.2% / scaleFactor | MODERATE | 2 |
| それ以外 | LOW | 1 |

**用途**:
- 動的閾値の計算に使用（後述）
- `volatilityFactor = max(0.3, min(2.0, ATRパーセント / 1.0))`

**重み**: 1.0（固定）

---

### 1.5 ROC（Rate of Change）

**目的**: 価格変化率の測定

**計算方法**:
```
期間 = 10 × periodScaleFactor
ROC = (現在価格 - n期間前価格) / n期間前価格 × 100
```

**シグナル判定**:
| 条件 | シグナル | Strength |
|------|----------|----------|
| ROC > 1.0% / scaleFactor | STRONG_UP | +10 |
| ROC > 0.3% / scaleFactor | UP | +5 |
| ROC < -1.0% / scaleFactor | STRONG_DOWN | -10 |
| ROC < -0.3% / scaleFactor | DOWN | -5 |
| それ以外 | NEUTRAL | 0 |

**重み**: 2.0〜2.3（短期時間枠で高め）

---

### 1.6 Market Sentiment（市場心理）

**目的**: 直近の価格変動の方向性を測定

**計算方法**:
```
期間 = 60 × periodScaleFactor（最小30）
上昇ティック比率 = 上昇した価格変動数 / 全変動数
平均変化幅 = |価格変動| の平均
```

**シグナル判定**:
| 上昇比率 | シグナル | 基本Strength |
|----------|----------|--------------|
| > 65% | BULLISH | +5 |
| < 35% | BEARISH | -5 |
| それ以外 | NEUTRAL | 0 |

**強度調整**:
- 平均変化幅が高い（HIGH）: Strength × 1.5
- 平均変化幅が低い（LOW）: Strength × 0.5

**重み**: 2.0〜2.5（短期時間枠で高め）

---

### 1.7 TimeframeTrendAnalyzer（期間分割評価）

**目的**: 分析期間全体のトレンド一貫性を評価

**計算方法**:
```
セグメント数:
- 15秒: 3分割
- 30-60秒: 4分割
- 180秒以上: 5分割

各セグメントで:
- 開始価格、終了価格
- 変化率 = (終了 - 開始) / 開始 × 100
- 方向判定: 変化率 > 閾値 → UP, < -閾値 → DOWN, それ以外 → FLAT
  （閾値: 30秒以下=0.02%, それ以上=0.05%）
```

**出力**:
- **Consistency（一貫性）**: 同一方向セグメント数 / 全セグメント数 × 100
- **Reversals（反転回数）**: UP↔DOWN の切り替わり回数
- **Reliability（信頼度）**: Consistency - (Reversals / (セグメント数-1)) × 50

**スコアへの影響**:
- 一貫性ボーナス: (Consistency / 100) × 15 点（方向に応じて加減算）
- 反転ペナルティ: Reversals × 5 点（減算）

---

### 1.8 EnhancedIndicatorCalculator（バランス調整MACD）

**目的**: 直近偏重を防ぎ、期間全体の動きを反映

**計算方法**:
```
期間を3分割:
- Early（前半1/3）
- Middle（中盤1/3）
- Late（後半1/3）

各期間でMACDを計算し、重み付け平均:
- weightedHistogram = Late × 0.4 + Middle × 0.3 + Early × 0.3
- weightedStrength = Late × 0.4 + Middle × 0.3 + Early × 0.3
```

**Consistency（方向一貫性）**:
- 全期間で同方向: 100%
- 2/3が同方向: 66%
- バラバラ: 33%

**重み**: 2.0（固定）

---

### 1.9 MultiScaleTrendAnalyzer（複数タイムスケール評価）

**目的**: 短期・中期・長期トレンドの整合性を確認

**計算方法**:
```
shortTerm = 直近 targetTimeframe 件のデータ
midTerm = 直近 targetTimeframe × 3 件のデータ
longTerm = 直近 targetTimeframe × 10 件のデータ

各スケールで:
- 変化率 = (終了 - 開始) / 開始 × 100
- 方向: 変化率 > 0.05% → UP, < -0.05% → DOWN, それ以外 → NEUTRAL
```

**Alignment（整合性）**:
- 3つ全て同方向: 100%
- 2つが同方向: 66%
- バラバラ: 33%

**スコアへの影響**:
- 整合性ボーナス: (Alignment / 100) × 20 点（短期の方向に応じて加減算）

## 2. スコア計算

### 2.1 総合スコアの計算

```
totalScore =
  MACD.strength × macdWeight
  + ADX.strength × 1.5
  + Stochastic.strength × 1.5
  + ATR.strength × 1.0
  + ROC.strength × rocWeight
  + Sentiment.strength × sentimentWeight
  + BalancedMACD.strength × 2.0
  + ConsistencyBonus（一貫性ボーナス）
  + AlignmentBonus（整合性ボーナス）
  - ReversalPenalty（反転ペナルティ）

maxScore = 各指標の最大スコアの合計

normalizedScore = (totalScore / maxScore) × 100
```

### 2.2 時間枠別の重み調整

| 時間枠 | macdWeight | rocWeight | sentimentWeight |
|--------|------------|-----------|-----------------|
| 15-30秒 | 1.2 | 2.3 | 2.5 |
| 60秒 | 1.5 | 2.0 | 2.0 |
| 180秒以上 | 1.8 | 2.0 | 1.5 |

### 2.3 感度係数（scaleFactor）

| 時間枠 | scaleFactor | 用途 |
|--------|-------------|------|
| 15秒 | 100 | 閾値を1/100に縮小（超高感度） |
| 30秒 | 50 | 閾値を1/50に縮小 |
| 60秒 | 20 | 閾値を1/20に縮小 |
| 180秒 | 5 | 閾値を1/5に縮小 |
| 300秒 | 1 | 通常感度（基準） |

### 2.4 期間係数（periodScaleFactor）

| 時間枠 | periodScaleFactor | 用途 |
|--------|-------------------|------|
| 15秒 | 0.25 | 指標の計算期間を1/4に |
| 30秒 | 0.5 | 指標の計算期間を1/2に |
| 60秒 | 1.0 | 基準 |
| 180秒 | 3.0 | 指標の計算期間を3倍に |
| 300秒 | 5.0 | 指標の計算期間を5倍に |

## 3. シグナル判定

### 3.1 動的閾値の計算

```
ATRパーセント = (ATR / 現在価格) × 100
volatilityFactor = max(0.3, min(2.0, ATRパーセント / 1.0))

// 仮想通貨の場合は追加調整
cryptoMultiplier = 仮想通貨ペア ? 1.5 : 1.0

highThreshold = 15 × volatilityFactor × cryptoMultiplier
strongThreshold = 50 × volatilityFactor × cryptoMultiplier
```

**通貨ペア別の閾値例**:
| 通貨ペア | 想定ATR% | volatilityFactor | highThreshold | strongThreshold |
|----------|----------|------------------|---------------|-----------------|
| USD/JPY | 0.3% | 0.3 | ±4.5 | ±15 |
| EUR/USD | 0.5% | 0.5 | ±7.5 | ±25 |
| EUR/JPY | 0.8% | 0.8 | ±12 | ±40 |
| GBP/JPY | 1.2% | 1.2 | ±18 | ±60 |
| BTC/USD | 6.0% | 2.0×1.5=3.0 | ±45 | ±150 |

### 3.2 trendConfidence（トレンド信頼度）

```
trendConfidence = ADX.strength × 1.5
// 範囲: 0〜15
```

### 3.3 シグナル判定ロジック

```javascript
if (normalizedScore > strongThreshold && trendConfidence > 7) {
  signal = 'STRONG_HIGH';
  confidence = min(95, 70 + normalizedScore / 3);
}
else if (normalizedScore > highThreshold && trendConfidence > trendStrengthThreshold) {
  signal = 'HIGH';
  confidence = min(85, 60 + normalizedScore / 4);
}
else if (normalizedScore < -strongThreshold && trendConfidence > 7) {
  signal = 'STRONG_LOW';
  confidence = min(95, 70 + |normalizedScore| / 3);
}
else if (normalizedScore < -highThreshold && trendConfidence > trendStrengthThreshold) {
  signal = 'LOW';
  confidence = min(85, 60 + |normalizedScore| / 4);
}
else {
  signal = 'NEUTRAL';
  confidence = null;
}
```

### 3.4 trendStrengthThreshold（ユーザー設定）

| 設定名 | 閾値 | 説明 |
|--------|------|------|
| 弱い | 3 | シグナル頻度優先（騙し多め） |
| 普通 | 4 | バランス型 |
| 強い | 5 | 精度優先 |
| 最強 | 7 | 高精度（シグナル少なめ） |

### 3.5 信頼度の調整

```javascript
// 一貫性・整合性による信頼度調整
reliabilityBonus = (segmentedTrend.reliability / 100) × 10
alignmentBonus = (multiScale.alignment / 100) × 10
reversalPenaltyPercent = min(segmentedTrend.trendReversals × 5, 20)

adjustedConfidence = confidence + reliabilityBonus + alignmentBonus - reversalPenaltyPercent
adjustedConfidence = max(60, min(95, adjustedConfidence))
```

## 5. 現在の課題と改善希望

### 5.1 現在の課題

1. **シグナル頻度が低い**
   - highThreshold=15が高すぎる可能性
   - trendConfidence条件が厳しい

2. **レンジ相場での精度**
   - 反転ペナルティが大きすぎる可能性
   - 横ばい相場でのシグナル精度が不明

3. **通貨ペア別の最適化**
   - 全通貨ペアで同じロジックを使用
   - ボラティリティ差を動的閾値で吸収しているが、十分か不明

### 5.2 改善希望

1. **精度の向上**
   - 騙しシグナルの削減
   - 勝率の向上

2. **シグナル頻度の増加**
   - 現在より多くの取引機会を得たい
   - ただし、精度を犠牲にしない範囲で

## 6. 評価いただきたい点

1. **指標の選択は適切か？**
   - 超短期取引（15秒〜5分）に適した指標か
   - 不足している指標、不要な指標はあるか

2. **各指標の重みは適切か？**
   - 時間枠別の重み調整は妥当か
   - 特定の指標を重視/軽視すべきか

3. **閾値設定は適切か？**
   - highThreshold=15、strongThreshold=50は妥当か
   - trendConfidence条件（ADX×1.5 > 閾値）は妥当か

4. **期間分割評価・複数タイムスケール評価は有効か？**
   - 一貫性ボーナス、整合性ボーナスの計算は妥当か
   - 反転ペナルティ（5点/回）は適切か

5. **改善提案**
   - 精度を維持しながらシグナル頻度を上げる方法
   - 追加すべき指標やロジック

## 7. 参考：シグナル判定フロー図

```
[価格データ入力]
       ↓
[10種類の指標計算]
       ↓
[重み付けスコア合算] → totalScore
       ↓
[正規化] → normalizedScore (-100〜+100)
       ↓
[ATRから動的閾値計算] → highThreshold, strongThreshold
       ↓
[ADXからtrendConfidence計算]
       ↓
[シグナル判定]
  ├─ normalizedScore > strongThreshold && trendConfidence > 7
  │    → STRONG_HIGH
  ├─ normalizedScore > highThreshold && trendConfidence > 閾値
  │    → HIGH
  ├─ normalizedScore < -strongThreshold && trendConfidence > 7
  │    → STRONG_LOW
  ├─ normalizedScore < -highThreshold && trendConfidence > 閾値
  │    → LOW
  └─ それ以外
       → NEUTRAL（見送り）
       ↓
[信頼度調整]
  - 一貫性ボーナス追加
  - 整合性ボーナス追加
  - 反転ペナルティ減算
       ↓
[最終シグナル出力]
  signal: HIGH/LOW/STRONG_HIGH/STRONG_LOW/NEUTRAL
  confidence: 60-95% または null
```
